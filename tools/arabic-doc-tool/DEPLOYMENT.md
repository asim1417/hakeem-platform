# نشر خدمة معالجة الوثائق (بايثون) — الدليل الكامل

هذه الخدمة تُشغّل **محرّك بايثون الحقيقي** (استخراج + OCR عربي + تنظيف) مع **معالجةٍ
في الخلفية** (ترفع → تُغلق الصفحة → تعود فتجد النتيجة)، و**Gemini كمزوّد إضافي اختياري**.

> **مهمّ:** لا تعمل على Vercel (لا يُشغّل بايثون/tesseract). تعمل على أي مستضيف Docker.
> واجهتك على Vercel تبقى كما هي وتُحيل المستخدم لهذه الخدمة (انظر «الربط» أدناه).

---

## الأسرع: حاوية Docker واحدة (أي مستضيف)

```bash
cd tools/arabic-doc-tool
docker build -t hakeem-doc-service .

# تشغيل — محلّي فقط (بلا Gemini، بلا قفل):
docker run -d -p 8080:8080 -v hakeem_doc_data:/data hakeem-doc-service

# تشغيل — مع Gemini كمزوّد إضافي + قفل بكلمة مرور:
docker run -d -p 8080:8080 -v hakeem_doc_data:/data \
  -e GEMINI_API_KEY="مفتاحك-من-Google-AI-Studio" \
  -e APP_PASSWORD="كلمة-مرور-قوية" \
  hakeem-doc-service
```

ثم افتح `http://عنوان-خادمك:8080`. البيانات تبقى في الحجم `hakeem_doc_data`.

## مستضيفات بنقرة (تدعم Docker)

| المستضيف | كيف | ملاحظة |
|---|---|---|
| **Railway** | «New → Deploy from repo» ويشير للمجلّد، أو ارفع الصورة | ~5$/شهر، أبسط خيار |
| **Render** | «New → Web Service» → Docker | خطّة مجانية للتجربة |
| **Fly.io** | `fly launch` في هذا المجلّد | حجم دائم `/data` |
| **VPS** (Hetzner/DigitalOcean) | ثبّت Docker وشغّل الأمر أعلاه | أرخص للأحجام الكبيرة |

على كلٍّ منها: اضبط متغيّرات البيئة `GEMINI_API_KEY` (اختياري) و`APP_PASSWORD` (مُستحسن)،
وأبقِ حجماً دائماً على `/data`.

---

## متغيّرات البيئة

| المتغيّر | الوظيفة |
|---|---|
| `GEMINI_API_KEY` | يُفعّل مزوّد Gemini الإضافي (اختياري) — بدونه يعمل المحلّي فقط |
| `APP_PASSWORD` | يقفل الأداة خلف تسجيل دخول (مُستحسن للنشر العامّ) |
| `JOBS_CONCURRENCY` | عدد الملفات المعالَجة معاً في المهمّة (الافتراضي 3، ارفعه لـ 6 للدفعات الكبيرة والمفاتيح المدفوعة) |
| `TOOL_DB` | مسار قاعدة البيانات (الافتراضي `/data/tool_docs.db`) |
| `JOBS_UPLOAD_DIR` | مجلّد ملفات المهام (الافتراضي `/data/job_uploads`) |

## واجهة الـ API (للربط أو الأتمتة)

| الطريق | الوظيفة |
|---|---|
| `GET /api/providers` | المزوّدون المتاحون: `{local:true, gemini:bool}` |
| `POST /api/jobs` | رفع ملفات (`files`) + `provider` (`local`/`gemini`) + `model` (`flash`/`pro`) → يبدأ مهمّة خلفية، يعيد `{job_id,total}` |
| `GET /api/jobs/{id}?text=1` | حالة المهمّة وتقدّمها (`done/total`) والنصوص — صالحٌ للاستطلاع بعد إغلاق الصفحة والعودة |
| `POST /api/jobs/{id}/import` | يستورد نتائج المهمّة إلى فهرس البحث |
| `GET /api/search?q=` · `GET /api/doc/{id}` | البحث والعرض |
| `GET /healthz` | فحص الصحّة |

**المعالجة في الخلفية:** المهمّة تعمل في خيطٍ على الخادم؛ إغلاق المتصفح لا يوقفها.
وإن أُعيد تشغيل الخادم، تُستأنف المهام غير المكتملة تلقائياً.

## الربط بمنصّة حكيم (Vercel)

بعد نشر الخدمة والحصول على رابطها (مثل `https://docs.example.com`):

1. في إعدادات مشروع Vercel أضِف متغيّر البيئة: `DOC_SERVICE_URL=https://docs.example.com`.
2. تعرض المنصّة زرّاً في صفحة الوثائق يفتح الخدمة الاحترافية (يظهر فقط عند ضبط المتغيّر).

هكذا: **واجهتك على Vercel، والمعالجة الثقيلة (بايثون + الخلفية + Gemini الاختياري) على الخدمة.**
