# محرّكات تحويل الوثائق العربية — الحزمة الكاملة

مأخوذة من منظومة الفحص القانوني (فرع `claude/zaini-case-audit-system-79pdof`)
**بعد تجريدها من كل ما يخص القضية**: لا أسماء أطراف، لا أرقام قضايا، لا بيانات —
محرّكات تحويل عامة تعمل على أي وثائق. فُكَّت الارتباطات القليلة بوحدات القضية
(ثوابت مسارات فقط) وأصبح الجذر قابلاً للضبط عبر `CASE_ROOT`.

## المحرّكات (scripts/)

### التفكيك والاستخراج
| السكربت | الوظيفة |
|---|---|
| `process_binaries.py` | فكّ أرشيفات zip/rar/7z واستخراج الثنائيات المعقّدة محلياً |
| `reocr.py` | إعادة OCR للـ PDF ذي طبقة النص التالفة — tesseract مباشرة على صور الصفحات |
| `reocr_hard.py` | المحرّك المعزّز: معالجة مسبقة للصور (تباين/حدة/تكبير) قبل tesseract |
| `reocr_generalize.py` | تعميم إعادة الـ OCR على مجموعات الملفات الضعيفة |
| `reocr_judgments.py` | نمط مخصص لصفحات الأحكام والمحاضر المزدوجة |
| `reocr_pilot.py` | تجربة سريعة على عيّنة قبل الدفعة الكاملة |
| `azure_engine.py` | OCR سحابي اختياري عبر Azure AI Document Intelligence — الإعداد من البيئة فقط، وتعطيل آمن بلا مفاتيح |
| `lift_engine.py` | استخراج مهيكل اختياري (Datalab lift): ‏schema-constrained + امتناع مدرّب — «لا تختلق نصاً» |

### الجودة والتنقيح
| السكربت | الوظيفة |
|---|---|
| `improve_readability.py` | إصلاح أسطر «الترتيب البصري» المعكوسة (أشكال تقديمية/تكرار) سطراً بسطر |
| `legal_clean.py` | تنقيح قانوني محافظ: إزالة ضوضاء الترويسات/التذييلات وضبط المسافات — لا حذف مضمون ولا إعادة صياغة |
| `qc_readability.py` | فحص جودة آلي: قياس بقايا الأسطر المعكوسة لكل ملف |

### التحويل والإخراج
| السكربت | الوظيفة |
|---|---|
| `make_word.py` | توليد Word ‏(.docx) عربي RTL من Markdown (عناوين/قوائم/جداول/اقتباسات) ومن CSV |
| `format_minutes.py` | تنسيق المحاضر إلى أقسام موسومة (بيانات الأطراف/الوقائع/…) — أنماط حقول عامة |

## المشغّلات

- `try_azure.py` — تجربة مقارنة فعلية: ضع `document.pdf` بجانب السكربت → يشغّل المحلي دائماً، وAzure إن كان مهيأً → `ocr_comparison_report.txt`
- `run_azure_test.sh` / `.ps1` — تشغيل ذاتي كامل للتجربة (‏Mac/Linux/Windows) بلا أي مفتاح في الكود

## التثبيت

```bash
cd tools/doc-conversion-engine
pip install -r requirements.txt
# أدوات النظام (للـ OCR وفك الأرشيفات):
sudo apt-get install -y tesseract-ocr tesseract-ocr-ara poppler-utils unrar p7zip-full
```

## الإعداد

انسخ `.env.example` إلى `.env`:
- `ENGINE_MODE=tesseract` (محلي، افتراضي) أو `azure`
- متغيرات Azure DI ‏(`AZURE_DI_ENDPOINT/KEY/MODEL`) — اختيارية، والغياب لا يُفشل شيئاً
- `CASE_ROOT` — جذر مجلدات العمل (`staging/`، `outputs/`) — افتراضياً مجلد الحزمة

## المبادئ المحفورة في المحرّكات

1. **لا اختلاق نص**: أي استخراج بالذكاء يمتنع (null) بدل التخمين
2. **تنقيح محافظ**: سلامة النص القانوني أولاً — لا حذف مضمون، لا إعادة صياغة
3. **لا أسرار في الكود**: كل الإعدادات من البيئة
4. **تراجع آمن**: غياب مكوّن اختياري (Azure/lift) لا يكسر المسار المحلي
