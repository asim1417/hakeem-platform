# حزمة الوكلاء المخصصين لحكيم — Hakeem Agent Kit

## تمهيد

هذه الحزمة تعرّف نظام الوكلاء المخصصين في منصة حكيم: البنية، والمخطط الملزم، ووكيل نموذجي كامل (التقاضي التجاري) بثلاث مهارات. الفكرة الحاكمة: **الوكيل المخصص طبقة تكوين فوق المحرك الموحد، لا محرك موازٍ**. ما يُخصص: النطاق، والقوالب، والنبرة، والمهارات. ما لا يُخصص أبدًا: التأريض، والنفاذ، وقيد النطاق، وبوابة التغطية، ومنع الترجيح الشخصي، والامتناع الصريح (ثوابت HLS‑001).

## محتويات الحزمة

| المسار | الدور |
|--------|-------|
| `schema/agent-manifest.schema.json` | المخطط الملزم لكل Manifest — الاعتماد توقيع عليه |
| `agents/commercial-litigator/manifest.json` | Manifest الوكيل النموذجي |
| `agents/commercial-litigator/AGENT.md` | شخصية الوكيل ونطاقه وثوابته |
| `agents/commercial-litigator/skills/scoped-research/` | مهارة البحث المقيد بالنطاق |
| `agents/commercial-litigator/skills/memo-drafting/` | مهارة إعداد المذكرة |
| `agents/commercial-litigator/skills/deadline-audit/` | مهارة حصر المُهَل (تطبيق HLS‑8.1) |

## البنية بثلاث طبقات

1. **المحرك الموحد (مشترك، مجمد):** `legal_research`، القراءة المتدرجة، المسح الشامل، مرحلة التحقق، بوابة التغطية. لا يملك أي وكيل تجاوزه.
2. **طبقة التكوين (Manifest):** ملف الممارسة + النطاق + المهارات + القوالب. هذا ما يُبنى لكل عميل.
3. **طبقة التسليم (MCP):** لكل وكيل endpoint خاص بمفتاح مصادقة، يكشف أدوات مسماة بممارسة صاحبه (`بحث_في_نطاقي`، `أعد_مذكرة`، `احصر_المهل`). الموصل يحمل هوية العميل، فكل استدعاء مقيد تلقائيًا بملفه ومعزول عن بيانات غيره.

## دورة حياة الوكيل

1. **مقابلة الممارسة:** نموذج مهيكل يستخرج: الدور، الأنظمة اليومية، المخرجات المعتادة، الجمهور.
2. **التجميع:** يُركب Manifest من المكونات الجاهزة — لا بناء من الصفر.
3. **الفحص الآلي:** تشغيل حالات المطابقة الست (HLS‑8.1 إلى 8.6) على الوكيل المجمع.
4. **الاعتماد:** مراجعتك وتوقيعك؛ ينتقل `approval.status` إلى `approved` مع قائمة الحالات المجتازة.
5. **النشر:** تفعيل endpoint الموصل بإصدار مرقم.
6. **التحديث:** أي تعديل يمس النطاق أو المهارات يرفع الإصدار ويعيد الفحص الآلي قبل النشر.

## كيف يُبنى وكيل جديد

1. انسخ مجلد `commercial-litigator` قالبًا.
2. عدّل `manifest.json`: الهوية، ملف الممارسة، النطاق، القوالب.
3. اكتب أو ركّب المهارات — كل مهارة تصرّح بأدوات المحرك التي تستدعيها في `engine_tools`، ولا تستدعي سواها.
4. اترك `hlsCompliance` كما هو — قيمه ثابتة `true` ولا يقبل المخطط غيرها.
5. مرر الوكيل على خط الاعتماد.

## أمثلة وكلاء قادمين على نفس القالب

| الوكيل | النطاق الافتراضي | مهاراته المميزة |
|--------|------------------|------------------|
| مستشار الامتثال | أنظمة القطاع المعني + اللوائح | فحص التزام، رصد تعديلات (`trace_amendments`) |
| الإدارة القانونية | الشركات + العمل + المنافسة | مراجعة عقد، رأي نظامي داخلي |
| مدقق الأحكام | المرافعات + المحاكم التجارية | فحص حكم، لائحة اعتراضية (منهجية أمان) |

## ملاحظة تنفيذ

هذه الحزمة تصميم مرجعي جاهز للتسليم إلى Claude Code. أدوات المحرك المذكورة (`legal_research`، `exhaustive_scan`…) هي المعرفة في «الأمر الجذري» و«الأمر التنفيذي الشامل»؛ فترتيب البناء الصحيح: المحرك الموحد أولًا، ثم طبقة الوكلاء هذه فوقه.
