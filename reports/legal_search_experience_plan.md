## 2) الفجوات الحقيقية (مرتّبة بالأثر مقابل المخاطرة)

### مجموعة أ — تحسينات واجهة عالية الأثر، منخفضة المخاطرة
1. **إعادة هيكلة صفحة المادة إلى تبويبات** (شرح/تحليل/أحكام/لائحة/فقه/أسئلة/مسائل/مراجع/سجل تعديلات) بدل بطاقات متناثرة — مع الفصل البصري الصارم (صندوق «النص النظامي الرسمي» منفصل عن «شرح تحليلي» عن «نصوص فقهية متوائمة غير ملزمة»).
2. **أدوات القراءة**: وضع القراءة، تكبير/تصغير الخط، تنقّل المادة السابقة/اللاحقة، نسخ الفقرة مقابل نسخ المادة بالإحالة.
3. **توسيع تبويبات صفحة النتائج** بأعداد لكل نوع + ترتيب (صلة/أحدث/رقم المادة) + «تضييق داخل نظام».

### مجموعة ب — البنية القانونية (متوسطة المخاطرة، ترحيلات مضافة فقط)
4. **حقول نصّ المادة الثلاثة**: `contentOriginal` (موجود كـ`content`) + `contentNormalized` + `contentPlain` + `sourceHash` + `confidenceScore` — ترحيل مضاف، تعبئة عبر سكربت.
5. **جداول الفقه المنضبطة**: `FiqhSource` + `FiqhText` + `FiqhArticleAlignment` (نوع الصلة، وجه الصلة، درجة الثقة، حالة المراجعة) — منفصلة تمامًا عن النظام، لا تُنشر إلا بعد مراجعة أو موسومة «مواءمة آلية غير معتمدة».
6. **المفاهيم والإحالات**: `LegalConcept` + `LegalArticleConcept` + `LegalCrossReference` (استخراج «وفقًا للمادة/مع مراعاة المادة» آليًا).

### مجموعة ج — الإدارة والـ API (منخفضة المخاطرة، إضافات)
7. **API بمسمّيات الأمر**: `/api/legal/*` (search/systems/articles/related/versions/fiqh/concepts/issues-tree) — أغلفة فوق المنطق القائم.
8. **لوحة إدارة المحتوى القانوني**: مراجعة الربط الآلي (مبادئ/فقه/إحالات)، soft-delete، سجل من-عدّل-ولماذا.

### مجموعة د — ضوابط الجودة
9. **اختبار يمنع خلط النص الفقهي بالنظامي** (وحدة + مكوّن).

## 3) ما لا يُنفَّذ (خارج النطاق الآمن أو يحتاج قرارًا)
- استبدال محرّك البحث بـMeilisearch/Elasticsearch — **مؤجَّل**: قرار المالك «تعزيز القائم فقط»، فنحسّن الترتيب على pgvector+trigram دون تبديل المحرّك.
- استيراد مصادر فقهية ضخمة جديدة — **مؤجَّل بقرار المالك**؛ نكتفي بتعزيز الطبقة القائمة (تراث الحيّ + FiqhIssue).
- **تصدير PDF/DOCX داخل النطاق** (لم يُؤجَّل): يُنفَّذ بأسلوب بلا اعتماد مكتبة (طباعة المتصفّح بنمط طباعة مخصّص + تصدير Word عبر blob HTML).

---

## 3.1) قرارات المالك المعتمَدة (تعديل الخطة)
- **الأولوية: الواجهة أولًا.**
- **الفقه: تعزيز القائم فقط** — لا جداول فقه جديدة؛ نضيف الفصل البصري الصارم + «وجه الصلة» + تنبيه «غير ملزم» + إظهار حالة المراجعة على طبقة تراث/FiqhIssue الحالية.
- **تأجيل استيراد الفقه الضخم.**
- **PDF/DOCX ضمن أدوات القراءة** (بلا مكتبة).
- **إبقاء محرّك البحث الحالي** وتحسين ترتيبه فقط.

---

## 4) خطة الدفعات المعدّلة (واجهة أولًا — كل دفعة: بناء + فحص + دفع + تقرير)

- **دفعة 1 (واجهة المادة) ⭐ نبدأ بها:** إعادة هيكلة صفحة المادة إلى تبويبات منظّمة + **الفصل البصري الصارم** للصناديق (نظامي/شرح/فقه) + أدوات القراءة (وضع قراءة، حجم الخط، السابق/اللاحق، نسخ الفقرة مقابل المادة بالإحالة) + **تصدير طباعة/Word بلا مكتبة**. لا تمسّ DB.
- **دفعة 2 (نتائج البحث):** ترتيب (صلة/أحدث/رقم المادة) + «تضييق داخل نظام» + عدّادات التبويب + توسيع فئات الاقتراح + تحسين ترتيب الصلة على المحرّك القائم.
- **دفعة 3 (تعزيز الطبقة الفقهية القائمة):** فصل بصري صارم لتبويب الفقه في صفحة المادة + «وجه الصلة» + تنبيه «نصوص فقهية متوائمة غير ملزمة» + إظهار المصدر/حالة المراجعة — على تراث/FiqhIssue، بلا جداول جديدة.
- **دفعة 4 (مفاهيم وإحالات — خفيفة):** استخراج الإحالات الداخلية («وفقًا للمادة/مع مراعاة المادة») وربطها، وإظهارها في الرأس الجانبي/المرتبطة. إضافي فقط.
- **دفعة 5 (API + إدارة + اختبار منع الخلط):** أغلفة `/api/legal/*` فوق المنطق القائم + لوحة مراجعة الربط الآلي + soft-delete + اختبار يمنع خلط النص الفقهي بالنظامي.

---

## 5) الضوابط الحاكمة الملتزَم بها في كل دفعة
- النصّ النظامي مصدر الحقيقة؛ لا يُعدّل ولا يُعاد صياغته.
- فصل بصري وبياني صارم: نظامي / لائحة / أحكام / شرح / فقه / آلي.
- الفقه طبقة مساندة غير ملزمة، بمصدر ووجه صلة وحالة مراجعة.
- لا نتيجة بلا مصدر/نوع/رقم/تاريخ عند توفّرها.
- لا كسر للمصادقة/المسارات/الجداول؛ الترحيلات مضافة فقط.
- الذكاء يشرح بعد عرض النصوص، يذكر مصادره، ويمتنع عند غياب النص.
