# تقييم مطابقة منصّة حكيم للمعايير (الحكومة الرقمية + القانونية + قواعد البيانات والبحث)
## standards_compliance_assessment.md — تحليل فجوات خبير

> التاريخ: 2026-06-23 · بقلم: مراجعة تقنية خبيرة (تحوّل رقمي + منصّات بحث قانوني + ترميز الأنظمة).
> **تنويه مهني:** هذا **تحليل فجوات خبير** لا شهادة مطابقة رسمية. المطابقة الرسمية لكود الحكومة الرقمية أو WCAG تتطلب تدقيقًا معتمدًا بأدوات قياس. كما أن **حكيم منصّة خاصة** (لا جهة حكومية)، فمعايير DGA الإلزامية تخصّ الجهات الحكومية؛ لكنها **مرجع ممارسات أفضل** ومهمّة إن استهدفت عملاء مهنيين/حكوميين.

## 1) كود الحكومة الرقمية (DGA) — بندًا بندًا

| المعيار | الحالة | ملاحظة |
|---|---|---|
| العربية أولاً + RTL كامل | ✅ مطابق | `lang="ar" dir="rtl"`، خطوط عربية احترافية |
| **ثنائية اللغة (عربي/إنجليزي)** | ❌ غير مطابق | لا إنجليزية ولا إطار i18n — معيار حكومي إلزامي |
| **الوصول الشامل (WCAG 2.1 AA)** | 🟠 غير مُدقَّق | لا تدقيق تباين ألوان، aria، تنقّل لوحة مفاتيح، قارئ شاشة |
| الهوية البصرية الموحّدة | ✅ جيّد | نظام تصميم موحّد (كحلي/ذهبي/رق) |
| التجاوب (جوال/تابلت/سطح) | 🟡 جزئي | تجاوب موجود؛ يحتاج اختبار أجهزة فعلي |
| الأداء (سرعة التحميل) | 🟢 جيّد | SSR + فهارس + حدود استرجاع |
| **النفاذ الوطني الموحّد (SSO)** | ❌ غير موجود | الحكومي يتطلب «النفاذ»/أبشر؛ حكيم: كوكي مخصّص |
| **حماية البيانات (PDPL)** | 🟠 جزئي/غير موثّق | تشفير مفاتيح + تدقيق؛ لكن لا سياسة خصوصية/موافقة/سجل معالجة موثّق |
| الأمان | 🔴 ثغرة | `DISABLE_AUTH=true` افتراضيًا (وصول مفتوح) — يجب `false` |
| سياسة الخصوصية + شروط الاستخدام | ❌ غير موجودة | صفحات إلزامية |
| البيانات المفتوحة (Open Data) | ⬜ غير مطبّق | تصدير معياري للبيانات القانونية |
| إمكانية التشغيل البيني (APIs موثّقة) | 🟡 APIs موجودة بلا توثيق رسمي (OpenAPI) |

**الأولوية الحكومية:** ثنائية اللغة + الوصول الشامل + النفاذ الوطني + PDPL موثّق + صفحات الخصوصية.

## 2) المعايير القانونية (هيئة الخبراء + ترميز الأنظمة)

المرجع: هيئة الخبراء بمجلس الوزراء هي **المصدر الرسمي** للأنظمة واللوائح، بترميز رسمي: اسم النظام، **رقم وتاريخ المرسوم الملكي**، تاريخ النفاذ، المواد، **التعديلات التاريخية**، الحالة (ساري/ملغى/معدّل).

| العنصر | الحالة في حكيم | الفجوة |
|---|---|---|
| الأنظمة + المواد | ✅ 489 نظام / ~16K مادة | جيّد |
| رقم المادة + النص | ✅ موجود | — |
| **رقم/تاريخ المرسوم الملكي** | 🟠 حقل `royalDecree` موجود لكن غير مملوء غالبًا | إلزامي للاستناد الرسمي |
| **تاريخ النفاذ** | 🟠 `effectiveFrom` موجود، غير مكتمل | مهم |
| **التعديلات والإصدارات (versioning)** | ❌ لا تتبّع للتعديلات التاريخية للمادة | فجوة جوهرية في المنصّات التشريعية |
| **حالة المادة/النظام** (ساري/ملغى/معدّل) | 🟠 `status="سارية"` افتراضي بلا إدارة فعلية | مهم |
| **الاستناد الرسمي الكامل** (نظام/مادة/مرسوم) | 🟡 جزئي (نظام+مادة) | يحتاج صيغة استناد رسمية موحّدة |
| الأحكام القضائية + ربطها بالمواد | ✅ 51K حكم + 29K رابط | متقدّم |
| المبادئ القضائية | 🔴 0 (غير مستخرجة) | فجوة |
| التصنيف الموضوعي | 🟡 كيان `LegalTopic` مُضاف، غير مملوء | يحتاج تعبئة |

**معيار عالمي مفقود:** لا يوجد **معرّف تشريعي دائم** (مثل ELI — European Legislation Identifier، أو Akoma Ntoso XML — معيار الأمم المتحدة لترميز التشريعات). هذه معايير المنصّات التشريعية المرجعية عالميًا.

## 8) خطة رفع المطابقة (مراحل)

- **عاجل (أمان/امتثال):** `DISABLE_AUTH=false` + سرّ قوي · صفحات خصوصية/شروط · توثيق PDPL (موافقة + سجل معالجة).
- **قانوني:** تعبئة أرقام/تواريخ المراسيم + حالة المادة · تتبّع التعديلات (إصدارات) · استخراج المبادئ · صيغة استناد رسمية موحّدة.
- **بحث/UX:** فلاتر متقدمة (محكمة/تاريخ/نظام/تصنيف/حالة) · شجرة هرمية للأنظمة · إكمال تلقائي · تمييز مطابقة كامل.
- **DGA/عالمي:** ثنائية اللغة (إطار i18n) · تدقيق WCAG AA · النفاذ الوطني · توثيق OpenAPI · معرّف تشريعي دائم (ELI/Akoma Ntoso).

## 9) رأيي الخبير (الخلاصة)

**حكيم منصّة قوية تقنيًا وتنافسية في العمق** — البحث الهجين (نصّي/دلالي/علائقي)، الربط الفعلي بـ51 ألف حكم و16 ألف مادة، وتكامل تراث، والإسناد المضبوط ضدّ الهلوسة — كلها تضعها **في مصافّ المنصّات الحديثة تقنيًا**، وأحيانًا أعمق من المنافس المحلي في الذكاء القانوني.

**لكنها ليست بعد «مطابِقة» بالمعنى الرسمي:**
- **حكوميًا (DGA):** تنقصها ثنائية اللغة، الوصول الشامل، النفاذ الوطني، وتوثيق PDPL — وهي إلزامية إن استهدفت القطاع الحكومي.
- **قانونيًا (هيئة الخبراء):** تنقصها **حوكمة المصدر الرسمي**: أرقام المراسيم، التعديلات والإصدارات، حالة النصّ، والاستناد الرسمي الكامل — وهي جوهر المصداقية في المنصّات التشريعية.
- **عالميًا:** لا تتبنّى بعد المعايير الدولية لترميز التشريعات (ELI/Akoma Ntoso) ولا الفلترة المتقدمة المعتادة في Westlaw/LexisNexis.

**توصيتي بالأولوية:** (1) تصليب الأمان فورًا، (2) حوكمة البيانات القانونية (مراسيم/تعديلات/حالة) لأنها أساس المصداقية، (3) الفلاتر المتقدمة + شجرة الأنظمة (أعلى أثر على تجربة الباحث)، ثم (4) متطلبات DGA (ثنائية اللغة/الوصول/النفاذ) إن توجّهت للحكومة.

**باختصار:** البنية التقنية ممتازة؛ الفجوة الحقيقية في **الحوكمة القانونية الرسمية والامتثال** لا في القدرة التقنية.
