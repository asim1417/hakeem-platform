# حالة استيراد الأحكام — موقوف عند قرار مواءمة قاعدة البيانات
## judgments_import_status.md

> التاريخ: 2026-06-21 · **لم تُنفّذ أي كتابة على الإنتاج.**

## ما حاولته
1. بحثت عن مصدر الأحكام في الحاوية: **لا ملف dump ولا `HOSTINGER_DB_URL`** (الحاوية طازجة؛ الأسرار في GitHub لا هنا).
2. حدّدت المسار الحقيقي: workflow **`migrate-judgments.yml`** (Hostinger MySQL → قاعدة سرّ GitHub) — **لم يُشغَّل ولا مرّة** (`total_count=0`).
3. شغّلت الفحص **القرائي الآمن** `inspect-vercel-db.yml` لمعرفة محتوى القاعدة الحيّة → **فشل خلال 9 ثوانٍ** عند فحص السرّ: **`VERCEL_DATABASE_URL_INSPECT` غير مضبوط** (لا اتصال بأي قاعدة).

## الخطر الذي أوقفني (قفل أمان مقصود في المستودع)
المنصّة على **ثلاث قواعد** (من `hakeem-db-environment-audit.md`):
- **A** قاعدة التشغيل الحيّة (Vercel/Neon) — ما يراه المستخدم.
- **B** سرّ `DATABASE_URL` في GitHub Actions (Supabase) — **هدف الـworkflow**، فيها **0 حكم**.
- **C** Hostinger MySQL — المصدر (~60 ألف حكم).

**A ≠ B.** فـ`migrate-judgments` يكتب في **B**، وقد لا يقرأها الموقع الحيّ. ولهذا وضع المهندسون قفل `NEON_RUNTIME_CONFIRMED` يمنع الكتابة حتى يؤكّد إنسان المواءمة.

**لماذا لم أشغّله:** لا أستطيع التحقق من المواءمة (سرّ الفحص غير مضبوط)، فلا يمكن الحصول على موافقة واعية. تشغيله أعمى قد يكتب 60 ألف صف في قاعدة خاطئة — إجراء يصعب التراجع عنه.

## الإجراءات المطلوبة منك لإكمال الاستيراد بأمان (اختر مسارًا)

### المسار 1 — تحقّق أولًا (موصى به)
أضف سرّ المستودع `VERCEL_DATABASE_URL_INSPECT` = رابط قراءة لقاعدة الموقع الحيّة.
→ أُعيد تشغيل `inspect-vercel-db` (قراءة فقط) فنرى كم فيها من أحكام فعلًا، ثم نقرّر.

### المسار 2 — إن كان سرّ `DATABASE_URL` في GitHub = نفس قاعدة الموقع الحيّ
أكّد ذلك، ثم أُشغّل `migrate-judgments`:
- عيّنة أولًا: `scope=sample` + `CONFIRM_RUNTIME_DB_ALIGNMENT=NEON_RUNTIME_CONFIRMED`.
- ثم الكامل: `scope=all` + `confirm=MIGRATE-ALL`.

### المسار 3 — مواءمة الأسرار أولًا
وجّه `DATABASE_URL` (سرّ GitHub) إلى قاعدة التشغيل الحيّة، ثم المسار 2.

## ما عدا ذلك: مُنجز ومُتحقَّق
كل الإجراءات البرمجية اكتملت (هوية/بحث شامل/كيانات DB/فهارس/أمان/تقارير، tsc=0، build 41/41). المتبقّي الوحيد هو **قرار تشغيلي + سرّ** لاستيراد بيانات الأحكام الحقيقية — لا يمكن إنجازه من داخل الحاوية ولا تشغيله أعمى على الإنتاج.
