## ١) القيد الحاكم على بيئة التنفيذ (إفصاح — بلا مبالغة)

بيئة التأليف **بلا قاعدة بيانات حيّة، ولا تطبيق مُشغَّل، ولا متصفّح**:
- `npm run typecheck` **يعمل** — وهو بوّابة التحقّق المعتمدة لكل دفعة (كلها خرجت بـ exit 0).
- `npm run lint` **غير مُهيّأ** (لا ملف ESLint) — خارج نطاق هذه الدفعات.
- `npm run build` **يفشل** في التوليد الساكن (صفحات ISR تنادي القاعدة عند البناء).
- **يتعذّر** التحقّق الزمني من الترتيب/الأداء (`eval:search`)، أو تخطيط الواجهة، أو `axe-core`.

**مبدأ القرار (وفق مبدأيك «لا تكسر ما يعمل» و«اختبر بعد كل خطوة»):**
أَشحن (وأدمج) فقط ما يمكن التحقّق منه ساكنًا بثقة عالية ومخاطر انحدار دنيا.
ما يُعيد كتابة ترتيب نواة البحث أو تخطيط التنقّل **لا يُدمج عمياءً**؛ يُوثَّق بدليل ومواصفة.

### التحفّظ الأمين الوحيد المتبقّي
- **الضبط الكمّي النهائي للترتيب** (أوزان/عتبات دلالية دقيقة) يُقاس بـ`npm run eval:search` (Recall@10/nDCG) على **قاعدة حيّة**. ما نُفِّذ في #275 هو **تصحيح مقياس مكسور** (اتجاهه صحيح بحكم البناء)، لا ضبط أوزان يحتاج معايرة. أي معايرة أوزان لاحقة تُجرى حيث تتوفّر Neon.
- الدفعات ٢.١/٣.١/٣.٢ تغييرات واجهة؛ يُستحسن تأكيد بصري سريع (معاينة) بعد الدمج — لا يُتوقَّع كسر (المكوّنات إضافية والمسارات محفوظة).

## ٤.٥) إصلاح نشر حَرِج — كان الإنتاج متوقّفًا (#276)

عند التحقّق من النشر تبيّن أن **بناء Vercel كان يفشل** (فلا شيء يصل الإنتاج — لا هذه الدفعات ولا ما قبلها):
- **الجذر (خلل سابق لا علاقة له بدفعاتي):** `instrumentation.ts` يُبنى لوقت **Edge** أيضًا، ويسحب `settings-service.ts` باستيراد `node:crypto` الساكن — غير المتاح في Edge (`Can't resolve 'crypto'` / `UnhandledSchemeError`). حارس `NEXT_RUNTIME` يمنع التنفيذ لا التحزيم.
- **الإصلاح:** تحميل `node:crypto` بكسلٍ مُخفى عن المُحزِّم (require غير مباشر) فلا يدخل رسم Edge؛ منطق AES-256-GCM بلا تغيير. `settings-service.ts` فقط (ليس ملف أمان مستبعَد).
- **التحقّق على Vercel الحقيقي:** معاينة #276 → **`success` (Deployment has completed)**؛ ومحليًّا `npm run build` → **`✓ Compiled successfully`** (كان يفشل عند الترجمة).
