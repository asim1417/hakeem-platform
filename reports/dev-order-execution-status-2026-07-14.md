# حالة تنفيذ الأمر التطويري الشامل — منصة حكيم

**التاريخ:** 2026-07-14 · **المنفّذ:** جلسة Claude Code (تفويض دائم من المالك أثناء غيابه)
**النطاق:** المسارات غير الأمنية (جودة البحث + التجربة + التوحيد). **المستبعد:** المصادقة/الدخول/IDOR/G1/G2/G11.

## القيد الحاكم على التنفيذ (إفصاح)
بيئة التأليف **بلا قاعدة بيانات حيّة ولا تطبيق مُشغَّل ولا متصفّح**. لذلك:
- **`npm run typecheck` يعمل** (بوّابة التحقّق الأساسية) — و`npm ci` نجح (454 حزمة).
- **`npm run lint` غير مُهيّأ** (لا ملف ESLint — مؤكَّد؛ خارج نطاق هذه الدفعات).
- **`npm run build` يفشل** في التوليد الساكن لأن صفحات ISR تنادي القاعدة عند البناء (لا DB).
- **لا يمكن التحقّق الزمني** من سلوك البحث/الترتيب أو تخطيط الواجهة أو `axe-core`.

**مبدأ القرار (وفق مبادئك #2/#3 «لا تكسر ما يعمل / اختبار بعد كل خطوة»):**
أشحن فقط الدفعات التي أستطيع التحقّق منها ساكنًا بثقة عالية ومخاطر انحدار دنيا. الدفعات التي تُعيد كتابة سلوك نواة البحث أو تخطيط الواجهة **لا أشحنها عمياء**؛ أوثّقها بمواصفة دقيقة لتُراجَع/تُنفَّذ مع وصول زمني.

## المواصفات الجاهزة للدفعات عالية القيمة (للتنفيذ مع وصول زمني)

### ١.٥ — الأوجه المبتورة (الملفات: `lib/modules/legal-core/comprehensive-search.ts`, `app/dashboard/legal-search/page.tsx`)
- المشكلة: `searchLegalCoreComprehensive(q, 30)` + سقوف 25/15/10 (`comprehensive-search.ts:23-25`) → الأوجه تُحسب على ≤50 صفًا (`legal-search/page.tsx:122-127`).
- الإصلاح الصحيح: أضف حقل `facets` اختياري لاستجابة `comprehensive-search` يُحسب على **المجموعة الكاملة قبل السقوف** (متاحة كـ`coreResults`/`hybridResults`)، وارفع جلب الأوجه لعيّنة أكبر (مثلاً 500) بمعامل منفصل `facetSampleSize`. القِس زمن الاستجابة قبل/بعد (`eval-search` P95).
- القبول: أعداد الفلاتر تطابق الإجمالي لا 30؛ لا ارتفاع P95 > 15%.

### ١.٧ — تصحيح إملائي (الملفات: `legal-retrieval.ts`, `lib/modules/legal-search/suggestions.ts`)
- عند 0 نتيجة، شغّل `SELECT ... word_similarity(term, title) > 0.3 ... ORDER BY similarity DESC LIMIT 5` (فهارس trgm موجودة). أعِد اقتراحات «هل تقصد؟».
- القبول: «الاثبت» → يقترح «الإثبات».

### ١.٤ — توحيد النواة + عتبة (الملفات: `comprehensive-search.ts`, `hybrid-search.ts`, `legal-retrieval.ts`)
- المجمّع مدخل وحيد ينادي النواة **مرة واحدة**؛ وحّد المقياس على RRF؛ عتبة تشابه دلالي دنيا 0.6؛ عتبة ثقة من التشابه الفعلي لا رتبة RRF.
- **إلزامي:** التقط baseline عبر `npm run eval:search` قبل التغيير، وقارن Recall@10/nDCG بعده. لا تدمج إن انخفضا.

*(بقية المواصفات مستخرجة من `reports/global-audit-2026-07-12.md` و`ux-legal-search-benchmark`.)*
