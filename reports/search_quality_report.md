## 1) المعمارية الحالية للبحث
- **منسّق هجين** `hybridSearch` يدمج أربعة مزوّدين قابلين للتبديل بلا كسر:
  | المزوّد | الحالة | الدور |
  |---|---|---|
  | PostgreSQL (نصّي) | ✅ فعّال | بحث النواة العربي (تفكيك/مشتقّات/جذر) للمواد + مطابقة كلمات للأحكام والمبادئ |
  | pgvector (دلالي) | ⚠️ معطّل | المتجهات فارغة + `SEMANTIC_SEARCH=false` |
  | الرسم المعرفي | ⚠️ معطّل | جدول العلاقات فارغ |
  | OpenSearch | ⬜ غير مُعدّ | جاهز للتفعيل عند الحاجة |
- **الدمج:** إزالة تكرار + تعزيز عند تعدّد المصادر + ترتيب بالصلة (0..1).
- **النطاق:** مادة + حكم + مبدأ، عبر **كل الأنظمة** (لا نظام واحد).

## 2) واجهة البحث الجديدة (`/dashboard/legal-search` — «البحث الشامل»)
رُقّيت من «صفحة اختبار» إلى **منتج بالهوية الموحّدة**:
- صندوق بحث مركزي بارز (Search-First) بأيقونة وحالة تركيز ذهبية.
- **فلاتر نوع الكيان** (الكل/المواد/الأحكام/المبادئ) مع عدّاد لكل نوع — تظهر **بعد** النتائج (بحث أولاً ثم تضييق).
- **بطاقات نتائج احترافية:** شارة نوع ملوّنة (مادة=كحلي، حكم=ذهبي، مبدأ=أخضر) + العنوان + **درجة الصلة %** + مقتطف + المصادر + **سبب الظهور** + زر **«فتح التفاصيل»** (مادة→صفحة المادة، حكم→صفحة الحكم).
- **شارات حالة المزوّدات** + عدد النتائج.
- **تنويه صريح** ببيانات الأحكام التجريبية الموسومة.
- **حالات موحّدة:** فارغ قبل البحث، لا نتائج، خطأ — كلها بالهوية ورسائل عربية واضحة.

## 3) نقاط الوصول الموحّدة للبحث
وُجّهت كل صناديق البحث إلى البحث الشامل:
- صندوق البحث في **شريط التنقّل العلوي** (كل الصفحات).
- **Hero الصفحة الرئيسية** للوحة التحكم.
- **بطاقة خدمة «البحث القانوني الشامل»** كأول خدمة في الرئيسية.
- عنصر **«البحث الشامل»** بارز أعلى الشريط الجانبي (بجانب الرئيسية).
