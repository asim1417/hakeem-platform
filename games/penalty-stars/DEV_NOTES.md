# DEV_NOTES — نجوم البلنتيات ⚽

## فحص المشروع الحالي

- **التقنية:** Phaser 3 (v3.80) + TypeScript صارم + Vite، مع Howler.js للأصوات وGSAP للحركات.
  البناء عبر `vite-plugin-singlefile` ينتج ملف HTML واحدًا مستقلًا (`dist/index.html`) يصلح
  للنشر على أي استضافة — منشور فعليًا على مسار `/penalty-stars` في منصة حكيم.
- **بنية اللعبة:** خمسة مشاهد Phaser (`src/scenes/`):
  - `BootScene` — توليد/تحميل الأصول (أشكال مرسومة + صور اللاعبين الحقيقية).
  - `MenuScene` — شاشة البداية والأزرار وخريطة المراحل.
  - `PlayerSelectScene` — بطاقات ٨ شخصيات.
  - `GameScene` — منطق اللعب كاملًا: السحب للتسديد (Arcade Physics)، الحارس، العداد، المذيع.
  - `ResultScene` — النتيجة بالنجوم ومنطق المراحل (اجتياز/بطولة/إعادة).
- **منطق اللعب:** في `GameScene.ts` — السحب من الكرة يحدد الاتجاه والقوة (بحد أقصى)،
  الحارس يرتمي بتخمين صحيح باحتمال محدود حسب الصعوبة، والحسم في `update()`.
- **البيانات:** `src/data/players.ts` (الشخصيات)، `src/config/gameConfig.ts` (الأبعاد،
  الألوان، الصعوبات، المراحل، العبارات).
- **الأصوات:** `src/utils/audio.ts` — مولّدة برمجيًا (WAV data URI) عبر Howler، قابلة
  للاستبدال بملفات mp3. **الصور:** أشكال مرسومة في `BootScene` + صور لاعبين في
  `src/assets/players/` (تُضمّن data URI عند البناء).
- **ما يمكن تطويره بأمان:** الخلفيات والكرة (طبقة رسم فقط)، بيانات اللاعبين، العبارات،
  إضافة مشاهد جديدة — دون لمس منطق التسديد/الحارس المجرَّب.

## خطة التعديل (منفذة في هذا التحديث)

1. توليد خلفيات ملاعب شبه واقعية (٥ ملاعب) وكرات شبه واقعية (٥ كرات) كأصول مضمّنة،
   مع إبقاء الرسم البرمجي القديم fallback إذا غابت الصور.
2. ملف `src/data/phrases.ts`: عبارات الهدف والمحاولة **وعبارات المدرب** الإرشادية.
3. أسماء الحراس حسب الصعوبة: فرفور (سهل) / صقر (متوسط) / أبو قفزة (صعب) مع عرضها في اللعب.
4. تحديث بيانات الشخصيات لمقياس ١–١٠ (سرعة/قوة/دقة) + نوع احتفال لكل لاعب
   (run/cup/fire/flag/crowd/stars/dance) وتنفيذ الاحتفالات بصريًا عند الهدف.
5. نظام تقدم بسيط بلا حسابات (`src/utils/progress.ts` + localStorage): مجموع النجوم
   يفتح كرات وملاعب — مشهد جديد `LockerScene` (🎒 الخزنة) للاختيار.
6. رسائل شاشة النتيجة حسب عدد الأهداف (٠–١/٢/٣/٤/٥) كما في المواصفة.
7. تحديث الواجهات لاستخدام الخلفيات الواقعية مع طبقة تعتيم خفيفة والعناصر المرحة فوقها.
8. بناء واختبار فعلي بالمتصفح بعد كل مجموعة تغييرات (`npm run build` + Playwright).

## الصور — الحالة والاستبدال

الصور الحالية **مولّدة برمجيًا** (شبه واقعية، مرحة، خفيفة) في `src/assets/images/`
عبر السكربت `scripts/generate-assets.py`. لاستبدالها بصور فوتوغرافية حقيقية ضع ملفات
بنفس الأسماء في `src/assets/images/` وأعد البناء (تُضمّن تلقائيًا في ملف اللعبة الواحد):

| الملف | المحتوى | المقاس المقترح |
|---|---|---|
| `stadium-real.jpg` | ملعب حقيقي نهاري | 720×1200 (طولي) |
| `stadium-school.jpg` | ملعب مدرسة | 720×1200 |
| `stadium-street.jpg` | ملعب حارة | 720×1200 |
| `stadium-stars.jpg` | ملعب ليلي بأضواء | 720×1200 |
| `stadium-cup.jpg` | ملعب نهائي الكأس | 720×1200 |
| `ball-real.png` | كرة حقيقية، خلفية شفافة | 256×256 |
| `ball-stars.png` / `ball-fire.png` / `ball-bolt.png` / `ball-gold.png` | كرات خاصة | 256×256 |

اضغط الصور (JPEG جودة ~72، PNG محسّن) — كل شيء يُضمّن في ملف واحد فلا تتجاوز
~150KB للملعب و~30KB للكرة.

ملاحظة الاتجاه: اللعبة **طولية (Portrait)** عمدًا — تجربة البلنتيات (مرمى أعلى، كرة أسفل)
أنسب للجوال بيد واحدة، و`Scale.FIT` يتكفل بالمتصفح الأفقي.

## الأمان (ثابت)

بلا إعلانات، بلا مشتريات، بلا دردشة، بلا تسجيل دخول، بلا روابط خارجية، بلا جمع بيانات —
التقدم يُحفظ محليًا فقط على جهاز الطفل (localStorage).
