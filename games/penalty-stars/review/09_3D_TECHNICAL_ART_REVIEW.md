# 09 — مراجعة 3D / 2.5D / Technical Art

## الوضع الحالي
اللعبة **2D بمنظور خلف نقطة الجزاء** فوق خلفية فوتوغرافية بمنظور حقيقي — أي أنها عمليًا «2.5D بالخداع البصري»: تصغير الكرة أثناء الطيران + ظل + خلفية perspective.

## هل تكفي 2D؟ — القرار
**نعم، مع ترقية 2.5D مقصودة. لا تنتقل إلى 3D الآن.** الأسباب:
- جمهور ٦-١٠ سنوات على جوالات متفاوتة: الويب الفوري (رابط، بلا تنزيل) ميزة توزيع حاسمة للعائلة، وUnity WebGL أثقل بكثير (≥20MB تحميل أولي) وأداؤه على جوالات الفئة المتوسطة غير مضمون.
- ميكانيكا البلنتي لا تستفيد من 3D إلا في الكاميرات؛ ويمكن محاكاة ٨٠٪ من الإحساس بتقنيات 2.5D أدناه.
- الفريق الحالي (شخص + مساعد) — ترحيل Unity = إعادة كتابة كاملة ٤-٨ أسابيع وفقدان كل الاختبارات، مقابل مكسب بصري يمكن تحقيق أغلبه بدونه.

## خطة ترقية 2.5D على Phaser (بالترتيب)
1. **قوس طيران للكرة**: بدل الخط المستقيم، أضف مكوّن ارتفاع وهمي `z` يرفع رسم الكرة عن ظلها (الظل يبقى على الأرض والكرة تعلوه ثم تهبط) — يعطي «تعليقة» ثلاثية مقنعة. جهد: يوم.
2. **حارس sprite بوضعيات** (12) مع تكبير طفيف عند الاقتراب.
3. **Parallax خفيف**: طبقة الجمهور تتحرك 2-3px عكس اهتزاز الكاميرا.
4. **جزيئات عشب** عند الركلة (Phaser ParticleEmitter — موجود بالمحرك).
5. **إضاءة**: هالات كشافات additive-blend فوق الملاعب الليلية.

## متى يُعاد فتح ملف Unity URP؟
فقط إذا أصبح الهدف: لعب حر 3D (مراوغات/مباراة كاملة بكاميرا تلفزيونية) أو نشر بمتاجر التطبيقات بمواصفات أعلى. عندها: Unity URP + Cinemachine + Input System + DOTween، بميزانية ترحيل واقعية ٦-٨ أسابيع لشخص متفرغ، ويظل إصدار الويب الحالي قائمًا كنسخة خفيفة.
