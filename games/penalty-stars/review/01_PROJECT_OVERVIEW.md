# 01 — تقرير الفهم العام للمشروع

## ١. التقنية المستخدمة

| السؤال | الجواب |
|---|---|
| Phaser؟ | **نعم — Phaser 3.80** (Arcade Physics + Scenes + Tweens) |
| Unity؟ | لا |
| React/Canvas؟ | لا — Canvas عبر Phaser فقط |
| TypeScript؟ | **نعم — صارم** (`strict: true`, `noUnusedLocals`) — صفر أخطاء `tsc` |
| محرك ألعاب حقيقي؟ | نعم للفئة المستهدفة: Phaser محرك ويب ناضج (مشاهد/فيزياء/إدخال/كاميرا) |
| مكتبات مساندة | Howler.js (صوت)، GSAP (حركات)، Vite + vite-plugin-singlefile (بناء بملف واحد) |

## ٢. هيكل المشروع (2478 سطر TS)

```
games/penalty-stars/
├── src/
│   ├── main.ts                 إعداد Phaser (480×800 طولي، Scale.FIT، Arcade)
│   ├── scenes/
│   │   ├── BootScene.ts        توليد القوام (حارس/نجمة/كرة/عدة UI) + تحميل الأصول
│   │   ├── MenuScene.ts        الشاشة الأولية (زر نيون + صف زجاجي + بطاقة اللاعب)
│   │   ├── PlayerSelectScene.ts شبكة ٢×٥ — اللمس يبدأ اللعب فورًا
│   │   ├── GameScene.ts        ⚠️ أكبر ملف (~650 سطر): تسديد/حارس/بطولة/مباراة/إعادة/مدرب
│   │   ├── ResultScene.ts      نتيجة الدور/المباراة + مكافآت
│   │   └── LockerScene.ts      الخزنة (كرات/ملاعب بالنجوم)
│   ├── config/gameConfig.ts    كل قيم اللعب (أبعاد/قوة/صعوبات/أدوار/ألوان/إصدار)
│   ├── data/
│   │   ├── players.ts          ١٠ لاعبين (إحصائيات ١-١٠، احتفال، صوت، صورة)
│   │   ├── phrases.ts          عبارات الهدف/المحاولة/المدرب/النتيجة
│   │   └── assetsManifest.ts   خريطة الأصول الموحدة
│   ├── utils/
│   │   ├── ui.ts               نظام الأزرار (نيون/زجاج/ذهبي) + رقاقات أيقونات
│   │   ├── audio.ts            ٨ مؤثرات مولّدة (WAV) عبر Howler
│   │   ├── announcer.ts        المعلق: ١٩ مقطع mp3 + فقاعة بصرية
│   │   ├── animations.ts       popIn/confetti/starBurst/احتفالات اللاعبين
│   │   └── progress.ts         التقدم محليًا (نجوم/كرة/ملعب/كأس)
│   └── assets/ (1.2MB)         ملعب حقيقي + ٤ مولّدة، ٥ كرات، ٧ صور أطفال، ٢١ صوت
├── scripts/                    generate-assets.py / generate-voice.py / fetch-assets.mjs
├── DEV_NOTES.md / ATTRIBUTIONS.md / ASSET_GUIDE.md / README.md
```

- **منطق التسديد**: `GameScene.setupInput/shoot/drawAimArrow` — سحب من الكرة، قوة بطول السحب.
- **منطق الحارس**: `GameScene.keeperDive/startKeeperIdle/update` — توقع نقطة الوصول + احتمال خطأ.
- **نظام بيانات منظم**: نعم — بيانات منفصلة عن المنطق، وقيم اللعب مركزية في `gameConfig.ts`.

## ٣. حالة المشروع الحالية

| البند | الحالة |
|---|---|
| اللعبة تعمل؟ | ✅ نعم — مختبرة بالمتصفح (Playwright) على كل التدفقات |
| أخطاء بناء؟ | ✅ لا — `npm run build` ينجح مع فحص TS صارم |
| أخطاء runtime؟ | ✅ لا أخطاء console في الاختبارات الآلية |
| Assets مفقودة؟ | ✅ لا — كل شيء مضمّن في ملف واحد (3.1MB) + fallback مرسوم لكل صورة |
| كود مكرر؟ | ⚠️ نعم — كتلة «خلفية الملعب + طبقة» مكررة في ٤ مشاهد؛ تُستخرج لدالة مشتركة |
| بنية قابلة للتوسع؟ | ⚠️ جزئيًا — البيانات والأدوات ممتازة، لكن `GameScene` أصبح god-file يجمع ٤ مسؤوليات (تسديد/حارس/مباراة/إعادة) ويحتاج تفكيكًا قبل إضافة الفاولات والمراوغات |

**الخلاصة**: أساس صلب يعمل وينشر، بنقطتي ضعف هيكليتين: حجم GameScene، وتكرار رسم الخلفيات.
