- **المسائل القانونية:** شجرة أقسام ← كتب (تنقّل هرمي فقط، بلا بحث نصّي).

### المزوّدات الأربعة للبحث الهجين — `lib/modules/legal-search/providers/`
1. **PostgreSQL** (`postgres-provider.ts`) — بحث نصّي كامل، متاح دائماً (قاعدة المنصّة).
2. **Vector** (`vector-provider.ts`) — دلالي pgvector، متاح (تغطية متجهات 100%)، بسقوط آمن للمعجمي.
3. **Knowledge Graph** (`knowledge-graph-provider.ts`) — يجد مواد مطابقة ثم يتوسّع عبر العلاقات للأحكام/المبادئ.
4. **OpenSearch** (`opensearch-provider.ts`) — فهرس خارجي اختياري؛ معطّل ما لم يُضبَط `OPENSEARCH_URL`.

## القسم ٨ — ملاحظات معمارية

1. **تدفّقات البحث:**
   - **استعلام DB مباشر** (الأنظمة، الأحكام، المبادئ): Prisma ORM بلا تحليل.
   - **بحث صرفي** (النواة): توسيع الاستعلام بقواعد عربية ثم مطابقة `LIKE`/`ILIKE`.
   - **بحث هجين** (الشامل): بحث متوازٍ عبر 4 مزوّدات، دمج وترتيب بالثقة.
   - **بحث مُسنَد** (RAG، الاستشارات): هجين ← اختيار سياق ← إسناد LLM ← إجابة.
   - **بحث الوكيل** (/ask): صرفي + استخراج مصطلحات + تنسيق متعدّد الأدوار + بثّ نتائج.

2. **توليد الأوجه (Facets):** الهجين يحسبها لحظياً عبر `facetsOf(rows, key)`؛ النواة بخيارات معرّفة مسبقاً؛ الصفحات الأخرى عبر `distinct`/`groupBy`.

3. **الصلاحيات:** كل صفحات/مسارات البحث تتطلّب `LEGAL_CORE_VIEW`؛ الاستشارات والقضايا تتطلّب `CONSULTATIONS_LIMITED`.
