# تنظيم وربط قواعد بيانات حكيم كمحرّك بحث قانوني عالمي
## معمارية البحث والمعرفة — خارطة طريق تنفيذية

> الهدف: رفع حكيم إلى مصافّ محرّكات البحث القانونية العالمية (Westlaw · LexisNexis · vLex)
> عبر تنظيم البيانات وربطها في **رسم معرفي موحّد**، ومسار استرجاع هجين، وترتيب بالسلطة والصلة،
> ومقياس جودة قابل للقياس. مبنيّ على أصول حكيم الفعلية (489 نظاماً · 15,902 مادة · أحكام · مبادئ · مكنز · مسائل).

## ١) ما الذي يميّز المحرّكات العالمية؟ (وأين حكيم منها)

| القدرة | المحرّكات العالمية | حالة حكيم |
|---|---|---|
| استرجاع هجين (معجمي + دلالي + مفاهيمي) | ✔ ناضج | ✔ موجود (BM25 + pgvector + مكنز + خريطة مفاهيم) |
| رسم معرفي يربط الكيانات | ✔ (KeyCite/Shepard's) | ◑ جزئي (روابط مواد↔أحكام، مسائل↔مواد) |
| **Citator / معالجة الاستشهاد** (هل المادة معدّلة/منسوخة؟ الحكم مؤيَّد/منقوض؟) | ✔ جوهري | ✗ ناقص — أكبر فجوة |
| ترتيب بالسلطة (الأكثر استشهاداً/الأعلى مرجعية) | ✔ | ✗ ناقص |
| فهم الاستعلام (مفاهيم/نية/مرادفات) | ✔ | ◑ خريطة مفاهيم + مكنز 2,967 |
| تصفية أوجه (Facets: نظام/مجال/محكمة/تاريخ) | ✔ | ◑ جزئي |
| مقياس جودة (مجموعات استعلام ذهبية) | ✔ أساسي | ✗ ناقص — أولوية قصوى |
| إسناد بلا هلوسة | — | ✔ ميزة حكيم (grounding guard) |
| بُعد فقهي مربوط بالأنظمة | — | ✔ ميزة حكيم حصرية |

**الخلاصة:** حكيم يملك بنية استرجاع قوية وميزتين فريدتين (الإسناد + الفقه)، وأكبر فجوتين هما: **معالجة الاستشهاد (Citator)** و**مقياس الجودة**.

## ٢) جرد الأصول الحالية (البيانات والروابط)

**الكيانات:**
- `legal_systems` (489) · `legal_articles` (15,902) · `amendments`/التعديلات
- `judicial_cases` (الأحكام) · `judicial_principles` (المبادئ)
- `legal_thesaurus_*` (مكنز: 2,967 مفهوماً · 26,749 موضعاً) · `glossary_terms`
- `fiqh_issues` (2,590) + `fiqh_issue_links` (7,768) — المسائل القانونية
- `embeddings` (pgvector، تغطية 100%) · فهرس BM25

**الروابط القائمة:**
- مادة ↔ حكم: `legal_article_case_links`
- مفهوم ↔ مادة: `legal_thesaurus_occurrences` (مواضع)
- مسألة فقهية ↔ مادة: `fiqh_issue_links` (مفتاح lawName+articleNumber)
- مفهوم ↔ نظام: `concept-map` (ترجيح الاسترجاع)
- علاقات عامة: `legal_relations` (knowledge-graph)

**الفجوة:** الروابط موجودة لكنها **غير موحّدة في رسم معرفي واحد قابل للاجتياز**، ولا يوجد **citator** (حالة المادة/الحكم) ولا **إشارات سلطة** (عدد الاستشهادات).

## ٣) المعمارية المستهدفة (5 طبقات)

```
┌─ طبقة الاستعلام ── فهم النية + توسيع مفاهيمي/مكنز + تصحيح إملائي عربي
├─ طبقة الاسترجاع ── هجين: BM25 ⊕ متجهات ⊕ مفاهيم ⊕ اجتياز الرسم المعرفي
├─ طبقة الترتيب ──── الصلة + السلطة (استشهادات) + الحداثة + ترجيح النظام الأصل
├─ طبقة المعرفة ──── رسم معرفي موحّد (كيانات + روابط + Citator/المعالجة)
└─ طبقة القياس ───── مجموعات استعلام ذهبية + مقاييس (P@k, MRR, nDCG, تغطية)
```

### ٣.١ الرسم المعرفي الموحّد (قلب التنظيم)
توحيد كل الروابط في نموذج كيان-علاقة واحد قابل للاجتياز:
```
النظام ──(يحتوي)──> المادة ──(عُدّلت بـ)──> التعديل
  │                   │
  │                   ├──(استُشهد بها في)──> الحكم ──(أرسى)──> المبدأ
  │                   ├──(تعالج)──> المسألة القانونية (فقه)
  │                   ├──(تَرِد فيها)──> مفهوم المكنز ──(تعريفه)──> الاستشهاد الرسمي
  │                   └──(حالتها)──> Citator: سارية | معدّلة | منسوخة
  └──(لائحته التنفيذية)──> نظام فرعي
```

### ٣.٢ معالجة الاستشهاد (Citator) — أكبر قيمة مضافة
لكل **مادة**: سارية/معدّلة/منسوخة + تاريخ النفاذ + المرسوم. لكل **حكم**: مؤيَّد/منقوض/مخالف + الأحكام اللاحقة التي تبعته أو خالفته. هذا ما يجعل المحامي يثق بالنتيجة (نظير KeyCite/Shepard's).

### ٣.٣ ترتيب بالسلطة
درجة النتيجة = `الصلة (BM25+دلالي) + السلطة (عدد استشهادات المادة/الحكم) + الحداثة + ترجيح النظام الأصل`. الأكثر استشهاداً يرتفع — مؤشّر مرجعية موضوعي.

### ٣.٤ مقياس الجودة (الأساس لكل تحسين ترتيب)
**مجموعة استعلامات ذهبية** (200–500 استعلام بأنظمتها/موادها الصحيحة) تُقاس آلياً (P@3, MRR, nDCG, تغطية الأنظمة). بدونها، أي ضبط ترتيب «تخمين». هذه **أولوية قصوى** — هي ما يحوّل التحسين من حدس إلى هندسة.

## ٤) خارطة الطريق المرحلية

### المرحلة ٠ — مُنجَزة اليوم ✅
- تصدير الأنظمة (485/15,902) + مزامنة `legal_systems` على Neon.
- تفعيل البحث الدلالي (متجهات 100%) + ربط المسائل القانونية (3,073).
- إزالة اقتطاع المرشّحين الأبجدي (تمثيل كامل للأنظمة).
- تحسين ترتيب الصلة (ترجيح النظام الأصل + 14 مفهوماً مدنياً).

### المرحلة ١ — مقياس الجودة + ضبط الترتيب (الأولوية القصوى)
- بناء **مجموعة استعلامات ذهبية** + سكربت قياس `eval:search` (P@k/MRR/nDCG).
- ضبط أوزان الترتيب (CONCEPT_SYSTEM_BONUS، الدلالي، الحداثة) على المجموعة — لا تخميناً.
- توسيع خريطة المفاهيم منهجياً من التشخيص.

### المرحلة ٢ — الرسم المعرفي الموحّد + Citator
- توحيد الروابط في خدمة اجتياز واحدة (`knowledge-graph`).
- بناء **Citator**: حالة كل مادة (من التعديلات/المراسيم) وكل حكم (من الإحالات).
- عرض «الحالة» في صفحة المادة/الحكم (شارة سارية/معدّلة/منسوخة).

### المرحلة ٣ — السلطة والتصفية والملاحة
- درجة السلطة (عدّ الاستشهادات) في الترتيب.
- تصفية أوجه كاملة (نظام/مجال/محكمة/تاريخ/حالة) في صفحة البحث.
- «المواد ذات الصلة» و«الأحكام التي طبّقت هذه المادة» (فهرس عكسي).

### المرحلة ٤ — النموّ
- اقتراحات الاستعلام (autocomplete) من المكنز.
- تنبيهات تحديث الأنظمة (Citator alerts).
- OpenSearch عند تجاوز الحدود (جاهز في الإعداد).

## ٥) لماذا هذا الترتيب؟
- **القياس قبل الضبط**: بلا مجموعة ذهبية، تحسين الترتيب يدور في حلقة تخمين (كما أظهر التشخيص: «15%» رقم مضلِّل يخلط الصلة بالانحياز).
- **Citator هو الفارق**: أكبر ما يميّز المحرّكات العالمية، وحكيم يملك مكوّناته (تعديلات/مراسيم/إحالات) غير مربوطة بعد.
- **السلطة تُرتّب بموضوعية**: تحوّل الترتيب من «تطابق نصّي» إلى «مرجعية قانونية».
