# ADR-000 — المعاون القضائي: خدمة مستقلّة، شريحة أولى بموصلٍ صناعيّ

- **الحالة:** مقبول
- **التاريخ:** 2026-07-19
- **السياق:** المواصفة النهائية v2.0 تطلب منتجًا قضائيًّا مستقلًّا (مساحة قضية، لا دردشة)، مع
  البدء بأوّل Sprint بعد فحص المستودع وتقرير الفجوات (§77).

## القرارات

1. **خدمة مستقلّة عن «اسأل حكيم».** مسار `/dashboard/judicial-assistant` ووحدة `lib/modules/judicial-assistant`
   منفصلان تمامًا عن `/dashboard/ask` والأوضاع الستّة. لا تعديل على منطق «اسأل حكيم».

2. **موصلٌ صناعيّ أوّلًا (لا تكامل وهميّ).** `CaseManagementConnector` عقدٌ ثابت، و`MockCaseConnector`
   يقدّم بياناتٍ صناعيّة موسومة صراحةً. لا يُفترض API لـ«تقاضي» (§75). عند توفّر النظام الرسميّ يُنفَّذ
   Adapter جديد خلف نفس العقد.

3. **بلا هجرة قاعدة بيانات في هذه الشريحة.** تقرأ الواجهات من الموصل الصناعيّ (ذاكرة)، فتُنشر على Vercel
   دون خطر «جداول غير مهاجَرة». نماذج Prisma القضائيّة تُضاف في المرحلة 1ب مع migration مستقلّ. التدقيق
   يستعمل جدول `AuditEvent` القائم.

4. **إعادة استخدام الاسترجاع المؤصَّل لـ JS-001.** `generateExecutiveSummary` يمرّ عبر
   `createAgentConsultationDraft` نفسه، فيرث الحجب الصادق وحارس أرقام المواد غير المؤصَّلة — التزامًا بـ«لا
   نصّ نظاميّ من الذاكرة» (§78) و«كلّ فقرة مدعومة» (§18.3).

5. **الصلاحية مؤقّتًا على `CONSULTATIONS_FULL`.** لا يوجد دور «قاضٍ» في RBAC الحاليّ؛ إضافته + ABAC
   (§12) مؤجّلة للمرحلة 1ب حتى لا تُلمس بذور الأدوار الآن. موثّقٌ كفجوة.

6. **كلّ مخرَجٍ مسودّة human-in-the-loop.** لا Auto-Approve؛ لافتة مراجعةٍ ووسم «تحتاج تثبيتًا بشريًّا»
   على كلّ نتيجة، واستشهاداتٌ قابلة للفتح.

## البدائل المرفوضة
- **اختصار المنتج إلى دردشة:** مرفوض صراحةً بالمواصفة (§78).
- **هجرة كامل النموذج القضائيّ الآن:** يرفع خطر النشر ويتجاوز نطاق أوّل Sprint.
- **دور «قاضٍ» + RLS فورًا:** يمسّ بذور الأدوار والمصادقة؛ يؤجَّل بقرارٍ صريح.

## الأثر
شريحةٌ عاملة قابلة للنشر تُثبت النمط (مساحة قضية تقودها المرحلة + عملٌ مؤصَّل + تدقيق)، وتترك
الاعتماديّات الخارجيّة (API رسميّ، قوالب أحكام، حقوق مصادر) مفتوحةً كما تقتضي §75.
