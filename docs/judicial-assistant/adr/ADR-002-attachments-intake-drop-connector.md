# ADR-002 — حذف موصل «تقاضي»؛ المدخل = مرفقات المستخدم، والقضية مشروع/مجلّد

- **الحالة:** مقبول
- **التاريخ:** 2026-07-19
- **السياق:** المخاطرة الأولى في سجلّ المواصفة (§74): «عدم توفّر API رسميّ (تقاضي) = احتمال مرتفع،
  أثر مرتفع». بناء المنتج على هذا الموصل يجعله معتمِدًا على بابٍ قد لا يُفتَح. الاستيراد اليدويّ
  المنضبط كان مذكورًا كاحتياط فقط.

## القرار
1. **حذف موصل «تقاضي» (`CaseManagementConnector` + `MockCaseConnector`) كأساس.** لم يَعُد المنتج
   يعتمد على أيّ نظامٍ خارجيّ. (يبقى ممكنًا لاحقًا كـAdapter اختياريّ لا يُبنى عليه الأساس.)
2. **المدخل الأساسيّ = مرفقات المستخدم.** القاضي يرفع وثائق القضية (لائحة، مذكّرات، محاضر)؛
   الاستخراج **محليّ في المتصفّح** (`extractFile`) — الملفّ لا يغادر الجهاز، يُرسَل النصّ فقط (PDPL).
3. **كلّ قضية مشروع/مجلّد يملكه القاضي** (`judicial_work_cases`، `owner_id`). ABAC حقيقيّ: تُرى
   للمالك أو المدير. الخريطة (أطراف/وقائع/…) تبدأ فارغة وتُثرى لاحقًا؛ المرفقات مادّة التحليل.
4. **الأعمال تعمل على المرفقات + النواة.** JS-001 وJS-018 يقرآن نصّ المرفقات (بسقفٍ للطول)
   ويؤصّلان بمواد وأحكام النواة. الحتميّة (JS-004/009/010) تعمل على الخريطة متى أُثريت.

## الأثر
- منتجٌ **مستقلّ عن أيّ API خارجيّ** — يعمل اليوم بمرفقات القاضي.
- هجرة القاعدة يدويّة idempotent: `npm run db:judicial` (يضيف `judicial_work_cases`). الوصول دفاعيّ
  (fail-open) فلا ينكسر النشر قبل تشغيلها (تظهر حالة فارغة مع رسالة).

## البدائل المرفوضة
- الإبقاء على الموصل كأساس: يُبقي المنتج رهينة اعتماديّةٍ مرتفعة الخطر (§74).
- تخزين الملفّات الخام في القاعدة: يُخزَّن النصّ المُستخرَج فقط (PDPL، أخفّ).

## يلغي جزئيًّا
ADR-000/ADR-001 (الموصل الصناعيّ) — يحلّ محلّه هذا القرار في طبقة المدخل؛ يبقى الباقي (الأعمال،
الدور، الحفظ) كما هو.
