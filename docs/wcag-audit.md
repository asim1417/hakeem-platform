# تدقيق الوصولية (WCAG 2.1 AA) — منصة الوثائق

**الأداة:** axe-core (فحص آلي معتمد) داخل Chromium حقيقي
**النطاق:** `‎/documents` (البوابة) · `‎/documents/tool` (البحث السريع) · `‎/documents/app` (محطة العمل)
**المعايير:** WCAG 2.0 A/AA + WCAG 2.1 AA
**التاريخ:** 2026-07-05

| الصفحة | المخالفات |
|---|---|
| البوابة | **0** |
| البحث السريع | **0** |
| محطة العمل | **0** |

## المخالفات التي كُشفت وأُصلحت في هذا التدقيق

| المخالفة | الخطورة | الموضع | الإصلاح |
|---|---|---|---|
| `link-in-text-block` — رابط داخل نص يعتمد على اللون وحده | جسيمة | رابط «محطة العمل» في ملاحظة البحث السريع | تسطير دائم للروابط داخل النصوص |
| `aria-required-children` — دور `list` بلا عناصر `listitem` | حرجة | قائمة نتائج الوثائق الفارغة في محطة العمل | الدور يُسنَد فقط عند وجود عناصر |
| `color-contrast` — تباين دون 4.5:1 | جسيمة | زر «تصدير» وعدّاد «المحدد» في محطة العمل | نحاسي أدكن `#b8874d`/`#7a5218` يحقق AA |

## عناصر الوصولية القائمة (من التصميم)

- بنية RTL كاملة مع `dir="rtl"` وخط IBM Plex Sans Arabic
- أسماء وصول للأيقونات التفاعلية (نظام HakeemIcons — `title` اختياري و`aria-hidden` للزخرفية)
- أهداف لمس ≥ 44px: دائمة في البحث السريع، وعبر `pointer: coarse` في شريط محطة العمل الكثيف
- `prefers-reduced-motion` يعطّل كل الحركات (الانتقالات، التلاشي، المؤشرات)
- حالات تركيز مرئية (`focus-visible`) على الأزرار والحقول
- `role="status"` لعدّادات ورسائل الحالة الحيّة

## إعادة الفحص

```bash
npm i --no-save playwright axe-core
node scripts/… # أو أعد تشغيل سكربت axe-audit على الصفحات الثلاث
```

> ملاحظة: الفحص الآلي يغطي القواعد القابلة للأتمتة؛ الفحوص اليدوية (قارئ صوتي فعلي،
> تنقّل لوحة مفاتيح كامل) تبقى ممارسة دورية موصى بها قبل الإطلاقات الكبرى.
