# تحقّق طبقة الاسترجاع في حكيم — ثمانية بنود بالرجوع إلى الكود

> تحويل فرضيّات خبير الاسترجاع إلى **ثغرات مؤكَّدة أو مشطوبة** بالقراءة المباشرة للكود.
> كل بند: الحكم · الدليل من الملف · القرار المترتّب.
>
> **أهمّ نتيجة:** بندان (٢ الأوجه و٧ التراث) انهارا لأنّ الجرد السابق — الذي بناه وكيل استكشاف على
> نموذج خفيف — تضمّن مزاعم **غير موجودة في الكود**. القراءة المباشرة صحّحتها.
>
> **تاريخ التحقّق:** 2026-06-30

---

## جدول الأحكام السريع

| البند | الحكم | الجوهر |
|---|---|---|
| ١. دمج الدرجات | 🔴 **مؤكَّد** | لا تطبيع ولا RRF؛ درجات حدسية بأرضية معجمية 0.55، دمج بـ `Math.max` |
| ٢. الأوجه (Facets) | ⚪ **مشطوب/مُصحَّح** | `facetsOf` غير موجودة أصلاً؛ صفحة الهجين صفحة اختبار بلا أوجه |
| ٣. التطبيع العربي | 🟡 **مؤكَّد بتحفّظ** | `ILIKE '%...%'` لكن مع فهرس trigram؛ تطبيع وقت الاستعلام بلا عمود مُفهرس |
| ٤. pgvector | 🟡 **منقسم** | فهرس HNSW موجود (يشطب «المسح الكامل»)؛ لكن التغطية غير مُتحقَّقة والاحتياطي معطوب دلالياً |
| ٥. ازدواج الحكم | 🔴 **مؤكَّد** | نفس بيانات `judicial_cases` بنموذجين؛ خطر ازدواج عند تفعيل `judgment` |
| ٦. الموجّه/التصنيف | 🔴 **مؤكَّد** | لا تصنيف قصد؛ منطق فهم مكرّر ومختلف بين الصناديق الثلاثة |
| ٧. مصدرية التراث | ⚪ **مشطوب** | لا وجود لمسار turath في الكود إطلاقاً |
| ٨. القياس | 🔴 **مؤكَّد** | لا golden set؛ `quality-gate` اختبار دخان لا تقييم استرجاع |

## ١. دمج الدرجات بين المزوّدات — 🔴 مؤكَّد (أول إصلاح)

**الملف:** `lib/modules/legal-search/hybrid-search.ts:93` — دالة `mergeResults` حرفياً:

```ts
const existing = map.get(key); // key = `${r.type}:${r.id}`
if (!existing) {
  map.set(key, { confidence: r.score, sources: [r.source], reasons: [r.reason], ... });
} else {
  existing.confidence = Math.max(existing.confidence, r.score);
  if (!existing.sources.includes(r.source)) existing.sources.push(r.source);
}
// لاحقاً لكل نتيجة:
if (m.sources.length > 1) m.confidence = Math.min(1, m.confidence + 0.05 * (m.sources.length - 1));
return [...map.values()].sort((a, b) => b.confidence - a.confidence).slice(0, limit);
```

- **لا تطبيع** (لا min-max، لا z-score، لا تحويل لرتب). **لا RRF** — بحث `rrf`/`reciprocal`/`1/(`/`k=60` ⇒ صفر.
- الدرجات **حدسية وغير متجانسة**:
  - مادة معجمية: `normalizeLexicalScore` يحشرها في **أرضية إجبارية `[0.55, 0.95]`** (`postgres-provider.ts:33`).
  - الأحكام: `0.58–0.9`. المبادئ: ثابتة `0.7`. المتجه: جيب تمام خام `[0,1]`.
- عند تعارض مزوّدَين على نفس الوثيقة: **`Math.max` لا جمع** (`:110`) + حافز اتفاق **0.05 فقط** (`:121`).

**الأثر:** الأرضية 0.55 ترفع المعجمي فوق دلالي قد يكون أدقّ (cosine 0.45)، والاتفاق بين مزوّدَين لا يُكافأ تقريباً.
**القرار:** الترتيب متحيّز بنيوياً ⇐ **RRF أو تطبيع حقيقي هو أول إصلاح**.

## ٢. حساب الأوجه (Facets) — ⚪ مشطوب ومُصحَّح (تصحيح نزاهة)

`facetsOf` **غير موجودة في الكود** (بحث على المستودع كله: صفر مطابقة).
و`app/dashboard/legal-search/page.tsx` الحقيقي **٩٩ سطراً فقط**، يصرّح عن نفسه:
«صفحة اختبار البحث الموحّد… **صفحة فنية للتحقق**». فيها **صندوق واحد** و**لا رقائق نوع،
لا فلاتر نظام/تصنيف/حالة/محكمة/سنة، لا ترتيب**.

> قسم «فلاتر البحث الهجين الديناميكية (8 أوجه)» في التقرير التشخيصي السابق كان **تلفيقاً من وكيل
> الاستكشاف** (استشهد بأسطر 199–300 في ملف لا يتجاوز 99 سطراً). القراءة المباشرة تنسفه.

**لا يوجد «عدّاد أوجه مغلوط» لأنه لا توجد أوجه أصلاً على هذه الصفحة.** الفلترة الحقيقية الوحيدة
في صفحات الأحكام/المبادئ عبر Prisma `distinct`/`groupBy` — أي على مستوى الفهرس، سليمة.

## ٣. التطبيع العربي ومكان حدوثه — 🟡 مؤكَّد بتحفّظ

- المطابقة النهائية **`contains` ⇐ `ILIKE '%...%'`** (`legal-retrieval.ts:246, 564-567`)، **لا `tsvector`** (`to_tsvector` ⇒ صفر).
- **لكن** توجد فهارس **GIN pg_trgm** على `content`/`title`/`judgmentText` (`scripts/sql/neon-retrieval-pre.sql:32-43`)
  ⇒ الـ`ILIKE` **مُسرَّع بالفهرس**، فليس «أبطأ حلقة».
- التطبيع (`normalizeArabicText`: همزة/تاء مربوطة/ألف مقصورة/تشكيل/تطويل) يحدث **وقت الاستعلام فقط**
  ويُوسَّع إلى **≤32 متغيّراً** (`buildArabicSearchVariants … slice(0, 32)`).
- **لا عمود مادّي `normalized_text` مُفهرس**؛ النصّ المخزَّن خام، والتطبيع يتكرّر مع كل استعلام.
- نمط «جذر» (`root`): استدلال يدوي (أول 3 أحرف بعد حذف و/ا/ي عبر `findRootCandidates`) **لا معجم جذور** ⇒ ضوضاء.

**الفجوة الحقيقية:** بحث متغيّرات مُطبَّعة ضدّ نصّ **غير مُطبَّع** ⇒ صور المصدر التي لا يولّدها أحد الـ32 متغيّراً **تُفوَّت** (فجوة استدعاء/recall).
**القرار:** ليست كارثة الأداء الموصوفة (trigram يكفي سرعةً)، لكن الإصلاح: **عمود مُطبَّع مادّي عليه GIN + تطبيع وقت الفهرسة**.

## ٤. تغطية ومسار pgvector — 🟡 منقسم

- **يُشطب «المسح الكامل»:** فهرس **HNSW** قائم —
  `idx_embeddings_hnsw_cosine … USING hnsw ("embedding" vector_cosine_ops)` (`scripts/sql/neon-retrieval-hnsw.sql`)،
  والاستعلام `ORDER BY embedding <=> q LIMIT` تقريبي ANN (شرط تطبيق السكربت على Neon).
- **يُؤكَّد «100% غير متحقَّق»:** لا فحص تغطية في الكود؛ `isAvailable` يكتفي بوجود **صفّ واحد**
  (`embeddingsTableHasRows` / `articleEmbeddingsExist` ← `findFirst`). الصفوف بلا متجه تُقصى صامتة (`WHERE embedding IS NOT NULL`).
- **عطب الاحتياطي:** عند غياب جدول `embeddings`، المسار الاحتياطي (`searchViaArticleEmbeddings`) يُرشّح
  **معجمياً أولاً** (`FALLBACK_CANDIDATE_POOL = 100`) ثم يحسب cosine ⇒ **لا يستطيع إظهار مطابقة دلالية بحتة
  فاتت المعجمي** — يُفرغ البحث الدلالي من غرضه في هذا المسار.
- **النموذج متطابق:** `text-embedding-3-small`/1536 فهرسةً واستعلاماً (`pre.sql:23` + `embeddings.ts`) ⇒ سليم ما دامت البيئة ثابتة.
- الفشل يُبتلع صامتاً على مستوى المزوّد (`embedText` يسجّل، لكن catch المزوّد صامت).

## ٥. ازدواج «الحكم»: كيان حيّ ومصدر معطّل — 🔴 مؤكَّد

- `judicial_cases` هو **الكيان الحيّ** (يُبحث عبر `postgres-provider` للأحكام + صفحة `judgments`).
- `judgment` **نوع مصدر معطّل** في `legal-core/search/page.tsx:150` (`<option value="judgment" disabled>حكم - لاحقًا</option>`)
  — خطّة لِطيّ الأحكام **داخل** خطّ المواد الموحّد (`searchLegalCore({ sourceTypes })`).
- أي **نفس البيانات الفيزيائية، نموذجان للاسترجاع**. تفعيله دون توحيد ⇒ **ازدواج**: الحكم يظهر مرّة
  من مزوّد الأحكام الهجين ومرّة من النواة.
- **تصحيح للجرد:** المعطّل **ثلاثة فقط** (`explanation`/`judgment`/`comparative`)؛ `hoqoqi_sql` **فعّال**
  (لا سمة `disabled` على `:148`) — لا أربعة كما ادّعى الجرد.

**القرار:** نفس المصدر ⇒ **وحّد النموذج قبل التفعيل**؛ `judicial_cases` هو مصدر الحقيقة.

## ٦. منطق الموجّه/التصنيف — 🔴 مؤكَّد (حجر الزاوية)

**الملف:** `app/api/ai/agent-search/route.ts:52-53`:

```ts
const numbers = (query.match(/\d+/g) ?? []).map(Number).filter(n => n > 0).slice(0, 3);
const terms = query.split(/\s+/).filter(w => w.length > 2).slice(0, 8);
```

- **لا تصنيف قصد** إطلاقاً؛ كل استعلام ⇒ `searchLegalCore({ searchType: "contains" })`.
- الاستشهاد المباشر («المادة ٧٧ من نظام العمل»): الرقم يُتحقَّق لاحقاً عبر `getArticlesByNumber(n)`
  كـ**خطوة تحقّق**، لا **مطابقة بنيوية (نظام + رقم) تتصدّر**. يُعامَل نصّاً حرّاً ويُترك للترتيب.
- **منطق الفهم مكرّر ومختلف** بين الصناديق:
  - `legal-search` ⇒ `hybridSearch`
  - `legal-core/search` ⇒ `searchLegalCore`
  - `agent-search` ⇒ تقطيع ساذج خاص ثم `searchLegalCore`

**القرار:** **نفس الاستعلام، نتائج مختلفة حسب الصندوق** ⇒ القياس مستحيل قبل توحيد طبقة الفهم.

## ٧. مصدرية التراث — ⚪ مشطوب (الفرضية باطلة)

بحث `turath` في كل `.ts/.tsx` ⇒ **صفر ملف**. لا `/api/turath/search`، لا `TurathSourcesPanel`، لا أي نقطة دمج.
الموجود **فقط** متغيّرات بيئة احتياطية في `.env.example` (`TURATH_API_BASE`/`TURATH_SEARCH_PATH`) كتجهيز مستقبلي.

⇒ لا دمج ولا سؤال «وزن سلطة» اليوم؛ ومزاعم الجرد عن مسار/لوحة تراث **ملفّقة**.
(لو دُمج مستقبلاً، يصبح وزن المصدرية/الحداثة قيداً واجباً يمنع تصدّر نصّ فقهي على نظام نافذ — لكنه الآن غير قائم.)

## ٨. الأساس المفقود — القياس — 🔴 مؤكَّد (العائق الأول)

لا `golden`/`P@`/`mrr`/`ndcg` في المستودع. الأقرب `scripts/quality-gate.ts` = **اختبار دخان**:

- يتحقّق `count == 9 أنظمة / 1981 مادة` (بذرة محلّية — لا 489 نظاماً على Neon).
- **5 استعلامات ثابتة** تُفحص فقط بـ`count > 0` أو `== 0` (لا حكم صلة، لا ترتيب).
- وجود الاستشهاد في `legal_articles` + عبارة المحاكاة الإلزامية.

**ليس تقييم ترتيب (IR eval).**
**القرار:** بلا مجموعة ذهبية، كل ضبط (وزن المفهوم، الدلالي، الحداثة) **تخمين**. يُبنى أولاً.

## الخلاصة التنفيذية وترتيب العمل

**بوّابات حرجة مؤكَّدة (١، ٣، ٦، ٨):** الدمج بلا تطبيع · التطبيع وقت الاستعلام · تشتُّت منطق الفهم ·
غياب القياس — هذه الأربعة تُبطل أيّ تحسين واجهة قبلها.

**الترتيب الصحيح:**
1. **(٨)** بناء `scripts/eval-search.ts` + مجموعة ذهبية أوّلية ← يجعل ما بعده قابلاً للإثبات.
2. **(١)** RRF / تطبيع حقيقي عبر المزوّدات الأربعة.
3. **(٦)** توحيد طبقة الفهم/التطبيع في خدمة واحدة مشتركة.
4. **(٣)** عمود مُطبَّع مادّي عليه فهرس GIN + تطبيع وقت الفهرسة.
5. **(٤، ٥)** تحقّق تغطية المتجهات فعلياً · توحيد نموذج الحكم قبل تفعيل `judgment`.

**مشطوب/مُصحَّح (٢، ٧):** لا عيب أوجه ولا عيب تراث — لأنهما غير موجودين أصلاً؛ الجرد السابق بالغ،
والقراءة المباشرة للكود هي المرجع.
