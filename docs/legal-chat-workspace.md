# منصة العمل القضائية الذكية (Legal AI Workspace) — الشات القضائي والمحاكاة

تحويل صفحة «المحاكاة القضائية» في حكيم من تحليل بضربة واحدة إلى **شات قضائي ذكي**
يفهم المستخدم أولاً، يعرض فهمه ويأخذ موافقته، يبني ملف قضية حيّ، يطبّق منطق الإجراءات
السعودية، ثم يصوغ بمنهج قضائي مُسنَد — مع منع الهلوسة ومراجعة بشرية.

> المسار الحاكم: **فهم → تأكيد → استكمال نواقص → استرجاع مصادر → تحليل → صياغة → مراجعة → حفظ/تصدير**

## المسارات
- الصفحة: `/dashboard/legal-chat` (أصبحت «المحاكاة القضائية» في القائمة).
- التحليل القضائي السريع (الصفحة السابقة) باقٍ على `/dashboard/judicial-simulation` باسم «التحليل القضائي السريع».
- الـAPI: `POST /api/legal-chat` (دورة شات)، `GET /api/legal-chat/conversations` (سجل المحادثات).

## المحركات (`lib/modules/legal-chat/`)
| الملف | الدور |
|---|---|
| `types.ts` | كل أنواع منصة العمل (نيّة، ملف قضية، بطاقات، مخرجات، حوكمة). |
| `taxonomy.ts` | قواميس كشف عربية حتمية (صفة/مسار/مرحلة/مخرج) + تسميات + تطبيع عربي. |
| `user-intent-engine.ts` | **UserIntentEngine**: فهم النيّة (حتمي أولاً + إثراء LLM اختياري عند الغموض). |
| `understanding-engine.ts` | **UnderstandingConfirmationEngine**: بطاقة الفهم + بوابة منع الإنتاج النهائي. |
| `case-file.ts` | **SimulationCaseEngine**: بناء/دمج ملف القضية الحيّ مع مصدر كل معلومة. |
| `judicial-procedure-engine.ts` | **JudicialProcedureEngine**: منطق إجرائي بحسب المسار (مرافعات/تجاري/جزائي/تحكيم/اعتراض). |
| `evidence-logic-engine.ts` | **EvidenceLogicEngine** + خطة إثبات/خريطة حجج/قائمة مسائل/خط زمني/ثقة/الخطوة التالية. |
| `drafting-style-engine.ts` | **DraftingStyleEngine**: قوالب الصياغة القضائية + عبارات معيارية. |
| `anti-hallucination.ts` | **SourceGroundingEngine + AntiHallucinationGuard**: إسناد من النواة ومنع اختلاق المواد. |
| `modes.ts` | الأنماط (10) + قوة البحث (5) + مكتبة الأوامر. |
| `role-simulations.ts` | الخصم الافتراضي + القاضي الافتراضي + المحكّم (منطق حقيقي من ملف القضية). |
| `document-analysis.ts` | تحليل المستندات (أطراف/تواريخ/مبالغ) + مراجعة العقود (بنود/مخاطر) + تعارض متعدد المستندات. |
| `redaction.ts` | إخفاء البيانات الحساسة (هوية/جوال/آيبان/حساب) — كامل/جزئي. |
| `workflows.ts` | LegalWorkflowEngine + LegalPlaybookEngine (مسارات قابلة للتشغيل). |
| `strategy.ts` | مقارنة الاستراتيجيات + «اشرح لماذا وصلت لهذه النتيجة». |
| `chat-orchestrator.ts` | المنسّق: يجمع كل المحركات ويوزّع البطاقات بحسب النمط/المخرج. |

## بطاقات الردّ
فهم الطلب · ملف القضية · المخرج (تصدير Word + إخفاء) · خطة الإثبات · خريطة الحجج · الخط الزمني ·
المسائل · درجة الثقة · **الخصم الافتراضي** · **القاضي الافتراضي** · **المحكّم** · **مراجعة العقد** ·
**تحليل المستندات** · **مقارنة الاستراتيجيات** · **اشرح النتيجة** · **مسار العمل** · الحوكمة.

## الحوكمة (مبادئ ملزمة في الكود)
1. لا مخرج نهائي عالي المخاطر قبل **بطاقة فهم + موافقة** (أو طلب «مسودة مع الافتراضات» صراحةً).
2. كل إسناد قانوني **من النواة فقط**؛ عند عدم وجود نص: تصريح صريح بلا اختلاق رقم مادة.
3. الافتراضات تظهر صراحةً داخل المسودات الأولية.
4. تنبيه تدريبي دائم + حالة مراجعة بشرية على كل مخرج.
5. لا تخزين محلي للبيانات القانونية (PDPL) — السجل في الذاكرة + حفظ خادمي best-effort.
