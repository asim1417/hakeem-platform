# 14 — جاهزية البيانات للربط والتصدير والتكامل (Word / Microsoft / الذكاء الاصطناعي)

فحص مركّز: **ما الذي يلزم لتُربَط بيانات حكيم وتُصدَّر وتتكامل** مع نظام إدارة خارجي أو موقع أو
Microsoft/Word أو محرّك ذكاء. مبنيّ على الكود (ملف:سطر)، بلا تعديل.

## الموجود فعلًا (نقاط قوّة للتكامل) ✅
| القدرة | الدليل |
|---|---|
| **بوّابة API عامة + OpenAPI 3.1 + CORS** (مفاتيح مُجزّأة، نطاقات) | `app/api/openapi/route.ts`، `gateway-auth.ts` |
| نقاط: مادة/متعلّقاتها/فقهها · بحث · أنظمة (قائمة مُصفّحة)/نظام | `app/api/legal/**` |

## أبرز الاحتياجات والنواقص (بالخطورة)

| الرمز | الخطورة | النقص | الأثر على التكامل | الدليل |

## أبرز الاحتياجات — مرتّبة (ما الذي يُمكّن كل سيناريو)

### لتكامل **محرّكات الذكاء / المواقع الخارجية**
1. **نقطة سحب جماعي + تغذية تغييرات** (`GET /api/legal/articles?updatedSince=…&cursor=…`) → مزامنة/فهرسة موثوقة. *(INT-001)*
2. **content_hash + مصدر رسمي** في كل عنصر → ثقة وكشف تغيّر. *(INT-003)*
3. (موجود ✅) OpenAPI + CORS + مفاتيح + llms.txt — أساس سليم.

### لتكامل **Word / Office**
4. **تعميم محرّك DOCX/PDF على المواد والأنظمة ونتائج البحث** (لديك المحرّك جاهزًا — يحتاج فقط ربطًا بنقاط `/api/legal`). *(INT-002)*
5. **استخراج نصّ Word/PDF المرفوع** (إكمال الـTODO) → إدخال مستندات Office. *(INT-004)*

### لتكامل **Microsoft / أنظمة الإدارة**
6. **أعمدة مصدر/إصدار/حالة منضبطة (enum)** → سجلّات قابلة للربط والفلترة. *(INT-003/006)*
7. **تصدير معياري (Akoma Ntoso أو JSON قانوني موثّق)** → تشغيل بينيّ. *(INT-005)*
8. **قناة نشر/مزامنة** بدل تخزين المرفقات فقط. *(INT-007)*

## خلاصة الجاهزية للتكامل: **3 / 5**
سطح API عام ومحرّك DOCX وموصّلات Microsoft/Drive وELI — أساس قويّ. لكن **التكامل الجادّ يحتاج**:
تغذية تغييرات + سحب جماعي (INT-001)، تعميم تصدير Word (INT-002)، وأعمدة مصدر/content_hash (INT-003).
هذه الثلاثة هي «أبرز النواقص» التي تفصل المنصّة عن التكامل السلس مع أي إدارة أو موقع أو ذكاء.

> ما تعذّر فحصه: سلوك الموصّلات الحيّ (SharePoint/Drive) يحتاج اعتمادات؛ وحجم المحتوى القابل للتصدير يحتاج القاعدة الحيّة.
