## 🔴 أ) قبل دمج إصلاحات P0 للإنتاج — إلزامي وإلا يُقفَل الموقع الحيّ
إصلاح SEC-001 يجعل **المصادقة إلزامية في الإنتاج**، وإصلاح SEC-006 يجعل **`AUTH_SECRET` إلزاميًّا**.
فإن دُمِجت بلا التجهيز التالي، **سيتوقّف الموقع الحيّ عن العمل** (لا أحد يدخل / يفشل الإقلاع).

- [ ] **١. ولّد سرّ الجلسة واضبطه في Vercel.**
  ```bash
  openssl rand -base64 32
  ```
  Vercel → Project → Settings → Environment Variables → أضِف `AUTH_SECRET` = الناتج (لكل من Production و Preview).
- [ ] **٢. أنشئ حساب مستخدم حقيقيًّا** (لأن دخول الزائر لن يعمل في الإنتاج):
  اضبط `INITIAL_ADMIN_EMAIL` و`INITIAL_ADMIN_PASSWORD` في Vercel ثم شغّل `db:seed`، **أو** أنشئ مستخدمًا عبر لوحة الأدمن قبل قفل الوصول.
- [ ] **٣. اضبط `DISABLE_AUTH=false`** (أو احذفه) في Vercel.
- [ ] **٤. جرّب تسجيل الدخول على رابط المعاينة (Preview)** للفرع قبل الدمج للإنتاج — تأكّد أنك تدخل بحسابك.
- [ ] **٥. بعد نجاح المعاينة فقط** → ادمج الفرع للإنتاج.

> بدون (١) و(٢): لا تدمج. أخبرني حين تجهّزها وأتحقّق معك من المعاينة.
