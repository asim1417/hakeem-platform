## القرار المختصر: **READY_WITH_CONDITIONS** (مشروط)

المحرّك والمحلّل العربي وworkflow الفهرسة **جاهزة كودًا**، لكن **أسس المزامنة والحوكمة
ناقصة** — الفهرسة اليوم «لقطة تُدفع مرّة» بلا آلية تحديث/حذف موثوقة.

## جدول التحقّق

| المتطلّب | الحالة | الدليل |
|---|---|---|
| مصدر حقيقة واضح في PostgreSQL | ✅ | Neon هو المصدر؛ OpenSearch مشتقّ (فهرس قراءة) |
| معرّف ثابت لكل وثيقة | ✅ | `_id = "${type}:${id}"` — `scripts/index-opensearch.ts:80` |
| `content_hash` | ❌ | لا يوجد عمود hash على `legal_articles` ولا في الفهرسة |
| `source_updated_at` | ⚠️ جزئي | يوجد `updatedAt` لكن لا يُفهرس ولا يُستخدم للمزامنة |
| `indexed_at` | ❌ | غير مُخزَّن (لا تتبّع لآخر فهرسة لكل وثيقة) |
| سياسة حذف/تحديث | ❌ | لا حذف للمحذوفات من الفهرس؛ إعادة الفهرسة تُضيف لا تُزامن |
| آلية مزامنة (Queue/Outbox/CDC) | ❌ | لا outbox ولا CDC؛ فهرسة يدوية بضغطة زر فقط |
| Retry / Dead-Letter Queue | ❌ | `_bulk` دفعة واحدة؛ لا إعادة محاولة للوثائق الفاشلة |
| Index Versioning | ❌ | اسم فهرس ثابت؛ لا `v1/v2` |
| Alias Strategy | ❌ | لا alias يفصل القراءة عن إعادة البناء |
| Zero-Downtime Reindex | ❌ | إعادة الفهرسة على الفهرس الحيّ مباشرة |
| Mapping صريح | ✅ | `ARTICLE_INDEX_BODY` / `CASE_INDEX_BODY` صريحة |
| Arabic Analyzer | ✅ | `hakeem_arabic` + `hakeem_arabic_exact` (تطبيع/تجذيع) |
| Metadata Filters | ⚠️ جزئي | `classification/status/eliSlug` مُفهرسة؛ لا فلترة تاريخ/نطاق |
| صلاحيات على الفهرس | ⚠️ | تعتمد اعتماد العنقود؛ لا تمييز أدوار داخل الفهرس |
| Tenant Isolation | ✅ N/A | لا بيانات مستخدمين تُفهرس (محتوى عام فقط) |
| Index Templates | ❌ | لا قوالب |
| Snapshot Policy | ❌ | لا نسخ احتياطي للفهرس (يُعاد بناؤه من Neon) |
| Monitoring | ❌ | لا مراقبة/تنبيه على صحّة العنقود من التطبيق |
| فلتر المراجعة صحيح | ❌ خلل | المزوّد يستبعد `status:"needs_review"` لكن قيمة الحقل ليست كذلك (SEARCH-004) |

## الشروط قبل اعتماد OpenSearch للإنتاج
1. **إضافة `content_hash` + `indexed_at`** على `legal_articles` لتمييز ما تغيّر.
2. **آلية مزامنة تزايدية** (outbox أو مسح `updatedAt > lastIndexed`) بدل الفهرسة الكاملة.
3. **حذف المحذوفات** من الفهرس (سياسة tombstone أو مسح دوري).
4. **Alias + إعادة فهرسة بلا توقّف** (`legal_articles_v2` ثم تبديل alias).
5. **تصحيح فلتر المراجعة** ليطابق الحقل/القيمة الصحيحة.
6. **Retry/DLQ** للوثائق الفاشلة في `_bulk`.

> بدون (1)-(3) يظلّ الفهرس يتباعد عن Neon بصمت مع كل تعديل — وهذا غير مقبول لمنصّة
> قانونية يجب أن يطابق بحثها النصّ الرسمي.
