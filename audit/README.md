# audit/ — أدوات تدقيق حكيم (قراءة فقط)

هذا المجلد يحوي **سكربتات فحص للقراءة فقط (Read-Only)** أُنشئت أثناء تدقيق أولي شامل
للمنصّة. **لا يُعدّل أيٌّ منها بيانات، ولا ينشئ/يحذف جداول أو فهارس، ولا يشغّل migration.**
جميع الاستعلامات `SELECT` فقط.

## لماذا سكربتات SQL بدل نتائج مباشرة؟
بيئة التدقيق **لا تملك اتصالاً بقاعدة الإنتاج (Neon)** — لا `DATABASE_URL` ولا شبكة.
لذلك جرد البيانات الحيّة (المرحلة الثالثة) وكثير من فحوص الجودة **تعذّر تنفيذها مباشرة**،
وحُوِّلت إلى استعلامات جاهزة تشغّلها أنت على القاعدة الحيّة.

## كيف تشغّلها (على القاعدة الحيّة، للقراءة فقط)
```bash
# مثال عبر psql (استبدل ب URL الحيّ — لا تضعه في الكود)
psql "$DATABASE_URL" -f audit/data-inventory.sql
psql "$DATABASE_URL" -f audit/orphan-records.sql
# ...إلخ
```
> يُفضَّل التشغيل على **نسخة/طرف قراءة (read replica)** إن توفّر. كل الملفات SELECT فقط،
> لكن الحذر واجب على قاعدة الإنتاج.

## الملفّات
| الملف | الغرض | يجيب على |
|---|---|---|
| `data-inventory.sql` | جرد شامل: عدد السجلات/الفارغة/بلا مراجعة لكل جدول | المرحلة ٣ |
| `orphan-records.sql` | سجلات يتيمة وعلاقات مكسورة ومراجع معلّقة | سلامة العلاقات |
| `duplicate-records.sql` | تكرار المواد/الأنظمة/الأحكام/العلاقات | عدم التكرار |
| `missing-required-fields.sql` | حقول جوهرية ناقصة في المواد | الاكتمال |
| `temporal-overlaps.sql` | تداخل/فجوات في نسخ المواد الزمنية | المرحلة ٥ |
| `current-version-conflicts.sql` | أكثر من نسخة «نافذة» للمادة نفسها | المرحلة ٥ (حرجة) |
| `source-validation.sql` | تصنيف المواد حسب موثوقية المصدر | المصدر |
| `content-anomalies.sql` | نصوص فارغة/قصيرة/مبتورة/رموز غريبة/أرقام غير موحّدة | صحة النصوص |
| `search-readiness.sql` | تغطية الفهارس والـembeddings وجاهزية البحث | المرحلة ٦/٩ |
| `rag-readiness.sql` | تغطية المصادر/الحالة/النسخ للـRAG | المرحلة ١١ |
| `opensearch-readiness.md` | قائمة تحقّق جاهزية OpenSearch (قرار) | المرحلة ١٠ |

## قراءة النتائج
كل استعلام موثّق بتعليق يشرح **الغرض** و**النتيجة المتوقّعة الصحّية** (ما الذي يعني أن
البيانات سليمة). أي رقم يخالف «المتوقّع» = بند تدقيق يُرفع في `audit/reports/`.

> **تنبيه:** أسماء الأعمدة camelCase في Prisma تُقتبس بعلامتَي اقتباس مزدوجتين في SQL
> (`"articleNumber"`)، والأعمدة المُعيَّنة بـ`@map` بأحرف صغيرة بلا اقتباس (`effective_to`).
